The CockroachDB cluster is fully operational. Here's a summary of what was diagnosed and done:

**Root cause:** The cluster was never initialized — all 3 nodes started correctly and were waiting for `cockroach init` to bootstrap the cluster. The early DNS errors for `crdb-cluster-1` were a transient race condition: `crdb-cluster-0` attempted to contact `crdb-cluster-1` while it was still in `ContainerCreating` (no IP yet), so DNS had no entry for it. By the time all pods were up, DNS resolved correctly (confirmed with `getent hosts`).

**What I fixed:**
- Ran `cockroach init --insecure --host=crdb-cluster-0.crdb-cluster.cockroachdb.svc.cluster.local:26257` from inside `crdb-cluster-0`

**Final state:**
- All 3 pods: `1/1 Running` (readiness probes passing)
- Node status: nodes 1, 2, 3 — all `is_available=true`, `is_live=true`
- SQL connections accepted; `SHOW DATABASES` returns system databases
- 3-node gossip confirmed via `crdb_internal.gossip_nodes`
