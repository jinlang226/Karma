The job runs successfully and prints "pong" with exit code 0.

**What was done:**

1. **Root cause**: The original script used `ray://ray-head:6379` — the `ray://` scheme triggers Ray Client mode (expects port 10001), which the `ray-head` service doesn't expose. Dropping the scheme to `ray-head:6379` fixed the address, but `ray.init(address=...)` requires the calling node to already be a registered Raylet in the cluster.

2. **Fix**: Updated the `ray-job-script` ConfigMap so `/opt/job.py` first calls `ray start --address=ray-head:6379` (joins the pod to the cluster as a worker node), then `ray.init(address="ray-head:6379")` succeeds and prints `pong`. The pod was recreated to pick up the ConfigMap change (subPath mounts don't hot-reload).
