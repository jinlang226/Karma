## Summary

A 3-member authenticated MongoDB replica set is now running in the `mongodb` namespace.

**Deployed resources**
- **StatefulSet `mongodb-replica`** — 3/3 pods ready (`mongodb-replica-0..2`), image `mongo:6.0`, each with a 1Gi PVC.
- **Headless Service `mongodb-replica-svc`** — `clusterIP: None`, governs the StatefulSet, giving each member stable DNS `mongodb-replica-<n>.mongodb-replica-svc.mongodb.svc.cluster.local`.
- **Secrets**: `mongodb-keyfile` (internal auth keyfile), `admin-user-password` and `app-user-password` (both key `password`).

**Configuration**
- mongod runs with `--auth --keyFile --replSet mongodb-replica`. An init container copies the keyfile and sets `chmod 400` / `chown 999` so mongod accepts it.
- Replica set `_id` is exactly **`mongodb-replica`** (confirmed via `rs.conf()._id`), not the `rs0` default.
- Members: `mongodb-replica-0` = **PRIMARY**, `-1`/`-2` = **SECONDARY**, all healthy.

**Users**
- `admin-user` / `admin123password` — `root` on `admin` (created via the localhost exception).
- `app-user` / `app123password` — `readWrite` on `appdb`.

**Data verification**
