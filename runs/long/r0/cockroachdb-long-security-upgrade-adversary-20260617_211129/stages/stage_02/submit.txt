Everything is working. Here's what happened and what was done:

**Root cause:** The cluster was never bootstrapped. CockroachDB requires a one-time `cockroach init` command to be run after all nodes start. The nodes were running and could reach each other (correct `--join` addresses in the StatefulSet), but they were stuck in a loop waiting for the init signal.

**Fix:** Ran `cockroach init --insecure --host=localhost:26257` on `crdb-cluster-0`, which bootstrapped the cluster.

**Current state:**
- All 3 nodes are `is_available=true` and `is_live=true`
- SQL is accepting connections (confirmed `SHOW DATABASES` and `SELECT version()` work)
- Cluster is v24.1.0 running in insecure mode
