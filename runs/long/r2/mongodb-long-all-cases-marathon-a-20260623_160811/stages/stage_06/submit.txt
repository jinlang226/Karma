TLS is fully enabled on the MongoDB replica set. Here's a summary of what was done:

1. **Generated certificates** using the `openssl-toolbox` pod:
   - CA key + self-signed certificate (`ca.crt`)
   - Server key + certificate with SANs for all 3 replica set member hostnames (`server.pem` = cert + key)
   - Client key + certificate (`client.pem` = cert + key)

2. **Created Secret** `mongo-tls` in the `mongodb` namespace with all three files.

3. **Patched the `mongo-rs` StatefulSet** to:
   - Mount the secret at `/etc/tls` (read-only)
   - Add `--tlsMode requireTLS --tlsCertificateKeyFile /etc/tls/server.pem --tlsCAFile /etc/tls/ca.crt` to the `mongod` command

4. **Verified**:
   - Plain connection → `MongoServerSelectionError` (rejected)
   - TLS connection with `/etc/tls/client.pem` + `/etc/tls/ca.crt` → `{ ok: 1 }` (succeeds)
   - All 3 replica set members are healthy (1 PRIMARY, 2 SECONDARY)
