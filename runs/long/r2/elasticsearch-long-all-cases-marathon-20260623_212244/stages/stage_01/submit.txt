All goals are met:

**Root cause:** The `es-config` ConfigMap was missing `cluster.initial_master_nodes`, which ES 7+ requires to bootstrap the initial master election. Without it, all nodes log "master not discovered yet" indefinitely.

**What was done:**

1. **Added `cluster.initial_master_nodes`** listing all 3 nodes to the ConfigMap.

2. **Restarted all 3 pods simultaneously** — required because the cluster had no state yet and needed all 3 to start together to reach quorum (2/3). The `subPath` mount means pods don't pick up ConfigMap changes without a restart.

3. **Cluster formed** — status green, `es-cluster-2` elected as master, all 3 nodes joined.

4. **Removed `cluster.initial_master_nodes`** from the ConfigMap — it's a bootstrap-only setting; leaving it in is dangerous because it can interfere with re-bootstrap scenarios.

5. **Rolling restart** (pod-2 → pod-1 → pod-0, one at a time) — each pod restarted with the clean config, rejoined using its persisted PVC cluster state, and cluster stayed **green** with 3 nodes throughout. A future rolling restart is safe to perform the same way.
