Everything is working correctly. Here's a summary of what was done:

**Setup:**
1. Generated a self-signed TLS certificate for `es.example.com` and created a Kubernetes secret `es-ingress-tls` in the `elasticsearch` namespace.
2. Created an `Ingress` resource with:
   - `ingressClassName: nginx` — uses the nginx ingress class
   - `tls` block referencing the `es-ingress-tls` secret — enables HTTPS at `es.example.com`
   - `nginx.ingress.kubernetes.io/backend-protocol: HTTPS` — because Elasticsearch itself listens on HTTPS (xpack.security.http.ssl)
   - `nginx.ingress.kubernetes.io/ssl-redirect: "true"` + `force-ssl-redirect: "true"` — any HTTP request gets a 308 permanent redirect to HTTPS
   - `proxy-ssl-verify: "off"` — skips verification of the self-signed backend cert

**Verification:**
- `https://es.example.com` → **401** (reached Elasticsearch, security prompt expected)
- `https://es.example.com` with credentials → **200** with valid cluster JSON response
- `http://es.example.com` → **308** Permanent Redirect (plain HTTP does not succeed)
