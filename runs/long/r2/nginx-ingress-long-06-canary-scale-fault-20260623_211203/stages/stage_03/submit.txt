Both routes work correctly:

- **Without header** → `stable`
- **With `X-Canary: beta`** → `canary`

**Root cause:** The `canary-canary` Ingress had `nginx.ingress.kubernetes.io/canary-by-header-value: always` instead of `beta`. The nginx-ingress canary feature matches the header value literally, so `X-Canary: beta` didn't trigger the canary rule. Changing the annotation to `beta` fixed the routing.
