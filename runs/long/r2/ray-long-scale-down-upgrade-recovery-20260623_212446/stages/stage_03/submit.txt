The command exits 0 and prints "pong". Here's what was done:

**Root cause:** The original script used `ray://ray-head:10001` (Ray Client protocol), but port 10001 is not exposed in the `ray-head` service (only 6379/GCS and 8265/dashboard are). Port 10001 is also not enabled on the head pod.

**Fix:** Updated the `ray-job-script` ConfigMap with a script that:
1. Calls `ray start --address=ray-head:6379` to join the cluster as a worker node — this is the correct way to connect to the GCS port 6379 from a client process
2. Then calls `ray.init(address="auto")` to connect to the local raylet that is now part of the cluster
3. Prints `"pong"` and shuts down cleanly

The pod was recreated to pick up the updated ConfigMap (subPath mounts don't hot-reload).
