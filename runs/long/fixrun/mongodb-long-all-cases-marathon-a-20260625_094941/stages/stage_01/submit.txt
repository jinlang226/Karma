**How it works:**
- All 3 pods start `mongod` with `--auth --keyFile` — members authenticate to each other using the shared keyfile
- Pod-0's init script uses MongoDB's localhost exception (no users → no auth required) to call `rs.initiate()` with all 3 members, then creates `admin-user` and `app-user`
- A marker file (`/data/db/.cluster_initialized`) prevents re-initialization on pod restarts
- Each pod is reachable at its stable DNS: `mongodb-replica-<n>.mongodb-replica-svc.mongodb.svc.cluster.local:27017`
