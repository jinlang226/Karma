All three pods are Ready and the replica set is stable with 1 PRIMARY and 2 SECONDARY members.

**Root causes fixed:**

1. **Missing `health-user` in MongoDB** — The readiness probe script authenticated as `health-user` but that user had never been created. Created it with the `clusterMonitor` role and password `healthpass123` (matching the `health-user-password` secret).

2. **Stale password override in `health-overrides` ConfigMap** — The configmap had an entry `mongodb-replica-1: legacyHealthPass123` which caused pod `mongodb-replica-1`'s health check to use the wrong password. Removed that stale override so the pod falls back to the correct password from the secret.

3. **Pod 0 running old spec** — Pod 0 was started from an older StatefulSet revision that lacked the health-check volumes and readiness probe entirely. The StatefulSet's rolling update reconciled it to the current spec once the other pods became Ready.
