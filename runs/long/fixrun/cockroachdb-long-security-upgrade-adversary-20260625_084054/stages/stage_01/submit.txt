All 3 nodes are `is_available=true` and `is_live=true`. The cluster is fully operational.

**Summary of what was deployed:**

| Resource | Name | Details |
|---|---|---|
| Headless Service | `crdb-cluster` | Selects on canonical labels, exposes grpc/http |
| Public Service | `crdb-cluster-public` | ClusterIP, same ports |
| StatefulSet | `crdb-cluster` | 3 replicas, `podManagementPolicy: Parallel`, image `v24.1.0` |
| PodDisruptionBudget | `crdb-cluster-pdb` | `minAvailable: 2` (quorum preserved during voluntary disruptions) |
| PVCs | `datadir-crdb-cluster-{0,1,2}` | 10Gi each, all Bound |

Each pod carries both required labels (`app.kubernetes.io/name: cockroachdb`, `app.kubernetes.io/instance: crdb-cluster`), runs in insecure mode, and advertises itself by its headless DNS name. The cluster was bootstrapped with `cockroach init` after all 3 nodes started in parallel.
