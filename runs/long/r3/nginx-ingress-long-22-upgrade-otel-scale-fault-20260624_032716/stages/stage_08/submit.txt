The fix is complete. Here's what was wrong and what I did:

**Root cause:** The `demo-ingress` was configured to use `expired-tls-secret`, which contained a TLS certificate valid only from 2020-01-01 to 2020-01-02 (already expired). Certificate validation failed immediately during the TLS handshake.

**Fix:**
1. Identified the correct CA (`/tmp/test-ca.crt`) — the one whose Subject Key ID (`F9:A8:...`) matches the Authority Key ID in the valid expired cert and matches the CA cert already loaded in the curl-test pod at `/tmp/tls/ca.crt`.
2. Generated a new TLS certificate for `demo.example.com` signed by that CA (valid 2026–2027).
3. Created the `demo-tls-secret` TLS secret with the new cert and key.
4. Patched `demo-ingress` to reference `demo-tls-secret` instead of `expired-tls-secret`.

Result: `curl --cacert /tmp/tls/ca.crt https://demo.example.com:31805/` returns exit code 0 and prints `hello`.
