- `node.name:` was set to null → Elasticsearch crashed immediately on startup with a parse exception
- `discovery.seed_hosts` pointed to `.default.svc.cluster.local` instead of `.elasticsearch.svc.cluster.local` → DNS resolution failed, so even if the node started it couldn't locate the cluster

**Fix:** Patched the ConfigMap to set `node.name: ${POD_NAME}` (Elasticsearch substitutes the `POD_NAME` env var already injected by the StatefulSet's downward API) and corrected the namespace in `discovery.seed_hosts`. Deleted the pod to force a restart since `subPath` mounts don't hot-reload.
