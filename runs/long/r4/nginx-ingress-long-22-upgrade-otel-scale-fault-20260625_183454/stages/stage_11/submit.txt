The rate limiting is fully operational. Here's the complete picture:

**nginx configuration (active):**
- Zone: `rate=1r/s` — hard limit of 1 request per second
- `limit_req ... burst=1 nodelay` — allows 1 burst request beyond the rate, then immediately rejects (no queuing)
- Status code for throttled requests: `503`

**Validation results:**
- `/api` with 10 requests at 0.5s intervals: **503** for excess requests (confirmed by nginx error logs showing "limiting requests, excess: 1.4 by zone ...")
- `/health` with same load: **all 200** (no rate limit on that Ingress)

```

nginx.ingress.kubernetes.io/limit-burst-multiplier: "1"
```

These translate directly to `limit_req_zone rate=1r/s` with `burst=1 nodelay` in the generated nginx config. The `/health` Ingress (`rate-health`) intentionally has no rate limiting annotations.
