All done. Here's what was wrong and what was fixed:

**Root causes:**
1. **Wrong password override** — the `health-overrides` ConfigMap had been applied with `mongodb-replica-1: legacyHealthPass123`, but the actual `health-user` password in MongoDB is `healthpass123`. This caused authentication failures on that pod's probes. The override data had been cleared without a clean `kubectl apply`, leaving a stale annotation.
2. **Probe timeout too tight** — both probes had `timeoutSeconds: 5`, but `mongosh` (Node.js runtime) needs more than 5 seconds to start, connect, and run the query under any load, causing spurious timeouts even when MongoDB was healthy.

**Fixes applied:**
- Re-applied `health-overrides` ConfigMap with empty `data: {}` so the annotation is consistent with the actual (no-override) state.
- Patched the StatefulSet to increase both `readinessProbe.timeoutSeconds` and `livenessProbe.timeoutSeconds` from `5` → `10`.

**Current state:** All 3 pods are `1/1 Ready`, replica set is `1 PRIMARY + 2 SECONDARY`.
