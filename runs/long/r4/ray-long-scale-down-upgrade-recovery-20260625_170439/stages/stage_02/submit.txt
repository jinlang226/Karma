The test returns HTTP 200. Here's what was done:

1. **Root cause:** The `ray start` command had `--include-dashboard=false`, which caused the dashboard to bind to `127.0.0.1` only — unreachable from other pods even with the Service port open.

2. **Deployment patch:** Replaced `--include-dashboard=false` with `--dashboard-host=0.0.0.0`, so Ray binds the dashboard to all interfaces, and added `containerPort: 8265` to the container spec.

3. **Service patch** (done first): Added the `dashboard` port (8265→8265) to `ray-head`.

After the rollout, `curl http://ray-head:8265/api/cluster_status` from inside the cluster returns 200.
