Both routes work correctly:

- **No header** → `stable`
- **`X-Canary: beta`** → `canary`

**Root cause:** The `canary-canary` ingress had `nginx.ingress.kubernetes.io/canary-by-header-value: always` instead of `beta`. In nginx-ingress, `always` is a special sentinel value that means "route to canary regardless of header value." Setting it to `beta` makes the controller only activate the canary when the header value matches exactly `"beta"`.
