A CockroachDB cluster is running in the cockroachdb namespace. One pod is
failing health checks and the cluster is degraded.

Your task:
Restore full cluster health so all pods are Ready and the cluster is healthy.

Agent task: Health Checking and Auto-Recovery
File: pkg/healthchecker/healthchecker.go

Health checking components:
1. Kubernetes readiness probe:
   - Checks /health?ready=1 endpoint
   - Pod removed from service if failing
   - Triggers pod restart if failing for too long

2. Liveness probe:
   - Checks /health endpoint
   - Kills pod if failing
   - Kubernetes restarts pod automatically

3. HealthChecker.Probe():
   - Used during rolling updates
   - Checks ranges_underreplicated metric
   - Waits until metric = 0 before continuing
   - Ensures cluster healthy after each pod restart

Health endpoints:
- /health: Basic liveness check
- /health?ready=1: Readiness check (considers cluster status)
- /_status/vars: Prometheus metrics including under-replicated ranges

Recovery process:
1. Readiness probe fails for one pod
2. Pod removed from Service (no new connections)
3. Operator waits for readiness to recover
4. Pod rejoins cluster and health checks pass

Verification:
1. Pod recovered:
   kubectl -n cockroachdb get pods
   (crdb-cluster-1 should show 1/1 READY)

2. All nodes in cluster:
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach node status --insecure
   (All 3 nodes should be is_live=true)

3. Verify SQL readiness:
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach sql --insecure -e "SELECT 1;"
   (Should succeed)
