Oracle passes with exit code 0. Here's what was done:

**Root cause:** The `expired-tls-secret` contained a leaf certificate valid only from Jan 1–2, 2020 — signed by the Test CA during precondition setup, but deliberately backdated to create the expired scenario.

**Fix:**
1. The precondition left the CA key at `/tmp/test-ca.key` (confirmed it matched the CA cert trusted by the `curl-test` pod at `/tmp/tls/ca.crt`)
2. Generated a new 2048-bit leaf key and CSR for `demo.example.com` using the existing `leaf.cnf` config (which includes the required SAN)
3. Signed the CSR with the CA key to produce a cert valid through 2028
4. Patched `expired-tls-secret` in-place with the new cert/key — the Ingress already pointed to this secret, so no Ingress change was needed
5. nginx-ingress picked up the renewed secret within ~5 seconds and HTTPS returned `hello`
