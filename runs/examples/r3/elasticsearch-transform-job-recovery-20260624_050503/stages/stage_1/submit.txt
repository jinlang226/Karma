Everything is working. Here's a summary of what was done:

**Root cause:** The `es-transform-config` ConfigMap had `node.roles: [ ingest ]` — the `transform` role had been removed as part of a capacity change, causing the transform node to join the cluster without transform capability.

**Steps taken:**
1. **Identified** that `es-transform` statefulset was scaled to 0 (no transform nodes at all), then scaled it back up.
2. **Diagnosed** the restarted node still lacked the `transform` role due to misconfigured ConfigMap.
3. **Fixed** `es-transform-config` to set `node.roles: [ transform, ingest ]` and restarted the pod.
4. **Created** the `events-by-service` transform via the `PUT /_transform/events-by-service` API (it never existed since the precondition's creation attempt failed with no transform nodes).
5. **Started** the transform via `POST /_transform/events-by-service/_start`.

**Result:** Transform is `started`, checkpoint advanced to 1, 5 source documents processed, destination index `app-events-rollup` now contains 2 aggregated records (`api`: 3 events, `web`: 2 events).
