**What was wrong and what was fixed:**

1. **Root cause:** The `expired-tls-secret` in the `demo` namespace held a TLS certificate valid only from 2020-01-01 to 2020-01-02 (expired ~5 years ago), causing all HTTPS connections to fail with certificate validation errors.

2. **Secondary issue:** `/tmp/ingress_env` had `INGRESS_NODE_IP=` (empty) because the precondition script didn't capture the node IP correctly. Fixed by writing `172.18.0.4` (the kind node's internal IP) into the file.

3. **Fix:** Generated a new 2048-bit RSA key and leaf certificate signed by the existing `Test CA` (from `/tmp/test-ca.crt` / `/tmp/test-ca.key`), valid 2025–2027, matching the original SAN (`demo.example.com`). Updated the secret in-place with `kubectl apply`. Nginx picked up the new cert within a few seconds and HTTPS now returns `hello` with exit code 0.
