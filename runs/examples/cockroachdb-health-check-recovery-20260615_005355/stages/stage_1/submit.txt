Cluster is fully healthy:

- **All 3 pods**: `1/1 Running` with 0 restarts
- **All 3 nodes**: `is_live=true`, `is_available=true`
- **SQL**: responding normally

**Root cause**: The StatefulSet startup command had been patched with a conditional that forced `crdb-cluster-1` to bind its HTTP server to `127.0.0.1:8080` (localhost only) instead of `0.0.0.0:8080`. The Kubernetes readiness probe hits the pod's cluster IP (`10.244.0.234:8080`), which is unreachable when the server only listens on loopback — so the probe failed, the pod was removed from the Service, and the cluster degraded.

**Fix**: Patched the StatefulSet to remove the per-pod conditional and use `--http-addr=0.0.0.0:8080` for all pods. The rolling update then recreated both `crdb-cluster-1` and `crdb-cluster-2` with the correct binding.
