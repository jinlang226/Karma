The replica set is fully healthy. Here's a summary of what was wrong and what was fixed:

**Root cause:** The `health-overrides` ConfigMap had a stale entry `mongodb-replica-1: legacyHealthPass123`. The health check script on pod 1 read this override file instead of the real password (`healthpass123` from the secret), causing SCRAM authentication to fail with "storedKey mismatch".

**Fix:** Removed the `mongodb-replica-1` key from the `health-overrides` ConfigMap. Pod 1 restarted (liveness probe had also fired), then came back up reading the correct password from `/etc/health/health-password`.
