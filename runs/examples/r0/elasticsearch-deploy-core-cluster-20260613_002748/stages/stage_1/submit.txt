The 3-node Elasticsearch cluster is fully deployed and operational:

**Resources created:**
- `ConfigMap/elasticsearch-config` — `elasticsearch.yml` with TLS, security, and discovery settings
- `Service/es-transport` (headless) — inter-node discovery on port 9300 (`publishNotReadyAddresses: true` so nodes can find each other before any are Ready)
- `Service/es-http` (ClusterIP) — HTTPS client access on port 9200
- `PodDisruptionBudget/elasticsearch-pdb` — keeps ≥2 nodes up during disruptions
- `StatefulSet/elasticsearch` (3/3 Ready) — `podManagementPolicy: Parallel` so all 3 nodes start concurrently and can bootstrap together

**Verification results:**
- Cluster health: `"status":"green"`, `"number_of_nodes":3`
- Write: `"result":"created"`, `"_shards":{"successful":2}`
- Read: 2 documents returned correctly

**Key decisions:**
- `publishNotReadyAddresses: true` on the transport service solves the bootstrap chicken-and-egg (pods can't become Ready until they form a cluster, but DNS for the headless service only resolves when pods are Ready)
- `podManagementPolicy: Parallel` ensures all 3 nodes start at once so the initial master election can proceed
- Transport SSL uses `verification_mode: certificate` (validates CA signature only, not hostname) since the pre-existing cert may not cover per-pod DNS names
