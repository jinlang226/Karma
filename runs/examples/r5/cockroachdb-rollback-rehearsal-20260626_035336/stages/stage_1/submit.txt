**What was found (non-default settings):**

| Setting | Current value | Default | Rollback action |
|---|---|---|---|
| `diagnostics.reporting.enabled` | `true` | `false` | `SET CLUSTER SETTING … = false` |
| `kv.snapshot_rebalance.max_rate` | `1.0 MiB` | `32 MiB` | `RESET CLUSTER SETTING …` |
| `RANGE default` zone | explicit override in place | inherited built-in | `CONFIGURE ZONE DISCARD` |
| StatefulSet replicas | 3 | 3 (matches creation spec) | no action needed |

**Excluded intentionally:**
- `cluster.secret` — auto-generated UUID, not user-configurable
- `version` — cluster version, cannot be downgraded via SQL
- System-range zones (meta, system, liveness, DATABASE system) — these are CockroachDB's own initialization defaults, not user overrides; discarding them would be destructive

The script is idempotent and includes a pre-flight connectivity check and a post-rollback verification step to confirm the settings landed correctly.
