**Root cause:** The `health-overrides` ConfigMap had a stale entry for `mongodb-replica-1` with password `legacyHealthPass123`. The health check script uses per-pod override files, so pod-1's health probe was authenticating with the wrong password instead of the current `healthpass123`.

**Fix:** Removed the stale `mongodb-replica-1` key from the `health-overrides` ConfigMap. The kubelet automatically refreshed the mounted volume, and the probe started passing. No data loss — the replica was always connected and replicating; only its readiness gate was broken.
