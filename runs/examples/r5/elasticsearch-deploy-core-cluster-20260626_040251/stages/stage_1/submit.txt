**Key design decisions:**
- `podManagementPolicy: Parallel` on the StatefulSet so all 3 pods start simultaneously (sequential would deadlock — es-1/es-2 wait for es-0, but es-0 needs the cluster quorum to pass readiness)
- `publishNotReadyAddresses: true` on the headless service so `es-headless` DNS resolves immediately for discovery, before any pod is Ready
- Same `es-http-tls` secret used for both HTTP and transport layers with `verification_mode: certificate` (skip hostname check, verify CA chain) — avoids needing a separate transport PKI
