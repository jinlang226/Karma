The CockroachDB cluster is fully operational. Here's a summary of what was done and why:

**Root Cause:** The 3 nodes were all running with `--join` flags pointing to each other, waiting in a mutual bootstrap deadlock — none of them could form the cluster without an explicit initialization command. This is expected CockroachDB behavior: peers can discover each other via `--join`, but someone must fire `cockroach init` to actually bootstrap the Raft consensus group.

**Fix:** Ran `cockroach init --insecure --host=localhost:26257` from `crdb-cluster-0`. This bootstrapped the cluster on node 0, after which nodes 1 and 2 joined automatically via the `--join` list they were already configured with.

**Final State:**
- All 3 pods: `1/1 Running` (readiness probe passing)
- All 3 nodes: `is_available=true`, `is_live=true`
- SQL connections accepted on port 26257 (public service `crdb-cluster-public` also available)
