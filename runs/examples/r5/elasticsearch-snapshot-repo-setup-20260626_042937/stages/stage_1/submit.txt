Everything is configured and working. Here's what was done:

**Root cause:** The S3 credentials in the `es-secure-settings` Secret were already loaded into the Elasticsearch keystore during pod initialization, but the keystore wasn't reloaded at runtime — so the credentials weren't active in memory yet.

**Steps taken:**

1. **Verified the setup** — the `repository-s3` module is built-in to ES 8.11.1 (not a plugin), the Secret's credentials were already in the keystore, and the S3 client endpoint/protocol were configured in `elasticsearch.yml` (no credentials there).

2. **Reloaded secure settings** via `POST _nodes/reload_secure_settings` — all 3 nodes confirmed successful reload with no restarts needed.

3. **Registered the repository:**
   ```json
   PUT _snapshot/minio-repo
   { "type": "s3", "settings": { "bucket": "es-backups", "client": "default", "path_style_access": true } }
   ```

4. **Created `snapshot-1`** — completed in 1.2 seconds with `state: SUCCESS`.

The snapshot `minio-repo/snapshot-1` is now stored in MinIO at `minio.elasticsearch.svc.cluster.local:9000/es-backups` with credentials sourced exclusively from the keystore (never from `elasticsearch.yml` or env vars).
