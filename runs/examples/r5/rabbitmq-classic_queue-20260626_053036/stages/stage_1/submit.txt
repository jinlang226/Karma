Everything is working. Here's a summary of the fix:

**Root cause:** `app-user` had no configure permissions on the `/app` vhost (configure field was empty). Queue declaration requires configure permission, so every attempt to create `app-queue` returned `Access refused`.

**Fix:** Granted full permissions to `app-user` on `/app`:
```
rabbitmqctl set_permissions -p /app app-user ".*" ".*" ".*"
```

**Verified outcome:**
- `app-producer` is Running (0 restarts on the new pod)
- `app-queue` in `/app` vhost is type `classic` with 16 messages
- All 3 RabbitMQ nodes remain healthy
