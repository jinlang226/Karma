**What was broken and how it was fixed:**

The precondition injected a fault by patching the `es-config` ConfigMap so that `discovery.seed_hosts` referenced the wrong namespace (`default`) instead of the actual namespace (`elasticsearch`). The correct DNS hostnames for the headless service are:
- `es-cluster-N.es-cluster.elasticsearch.svc.cluster.local`

When es-cluster-2 was restarted (after its PVC was deleted), it couldn't find any seed hosts to bootstrap discovery, so it couldn't rejoin the two-node cluster that was already running.

The fix patched the ConfigMap to use the correct `elasticsearch` namespace in all three seed host entries, then deleted es-cluster-2 so the StatefulSet controller recreated it with the corrected config. On restart, it successfully discovered the existing nodes and rejoined the cluster.
