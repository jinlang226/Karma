The `config-audit` ConfigMap is created with a 12,014-character findings report. Here's a summary of what was found:

**Critical**
- **C-1**: No TLS on AMQP (5672) or Management API (15672) — all traffic is plaintext

**High**
- **H-1**: RabbitMQ 3.12.14 is explicitly flagged "out of support" by the broker itself — no future security patches
- **H-2**: `app-queue` is a classic (non-replicated) queue with no HA policy — a single node failure makes it unavailable

**Medium**
- **M-1**: No per-user connection/channel limits on either user
- **M-2**: `app-user` has full `configure=.*` permissions on `/app` — not least-privilege
- **M-3**: No TTL, dead-letter exchange, or overflow policies on any queue

**Low**
- **L-1**: Default vhost `/` is retained but unused
- **L-2**: Management API listening on all interfaces without a NetworkPolicy

Cluster state was not modified — all operations were read-only (`rabbitmqctl list_*`, `cat` on config files, `get`/`describe` on Kubernetes resources).
