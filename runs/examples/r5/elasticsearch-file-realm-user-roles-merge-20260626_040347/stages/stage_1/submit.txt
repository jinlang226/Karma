The oracle passes. Here's what was done and why:

**Root cause:** The `report-user` existed only in the `user-provided-file-realm` secret, but Elasticsearch's StatefulSet was configured to load the file realm exclusively from `es-file-realm-aggregate` (mounted via the `file-realm` volume). The aggregate only contained `ops-user`.

**Fix:**
1. Merged `report-user`'s `users` and `users_roles` entries from `user-provided-file-realm` into `es-file-realm-aggregate`.
2. Triggered a rolling restart of the `es-cluster` StatefulSet — necessary because the volume files were mounted with `subPath`, which prevents Kubernetes from auto-updating them when the secret changes.

**Result:** All 3 nodes restarted cleanly (cluster remained green throughout), both `ops-user` and `report-user` are now in the file realm, and `report-user` can read the `app-data` index with the `viewer` role.
