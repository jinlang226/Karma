The command exits 0 and prints "pong". 

Here's what was done:

- **Root cause:** `ray.init(address="ray-head:6379")` uses the GCS (cluster-internal) protocol. This requires a local raylet process to be running on the client — without it, Ray looks for a local session file that doesn't exist and times out.

- **Fix:** The updated script calls `ray start --address=ray-head:6379` first to spin up a local raylet that joins the cluster, then `ray.init(address="ray-head:6379")` succeeds because the local session exists.

- The ConfigMap `ray-job-script` was patched with the new script, and the `ray-client` pod was recreated (subPath mounts don't hot-reload, so a pod restart was needed).
