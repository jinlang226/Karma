**The migration plan (`plan.md` in the `change-plan` ConfigMap) covers 8 changes toward the hardened baseline:**

1. **Disable SCRAM-SHA-1** — retain SHA-256 only
2. **Least-privilege user roles** — split the single super-account into per-function accounts
3. **Enable TLS** (`allowTLS` → `requireTLS` two-phase rolling restart)
4. **Restrict network binding** + apply a NetworkPolicy on port 27017
5. **Lower log verbosity to 0** — live `setParameter`, no restart needed
6. **Tighten slow-op threshold to 50 ms** — live `setParameter`, no restart needed
7. **Migrate journal compressor to `snappy`** — requires rolling initial sync per member
8. **Raise default write concern to `majority`** — live `rs.reconfig()`

The plan includes an execution ordering table with dependencies, rollback notes for each change, and explicit "do not apply now" callouts. The live cluster was not modified.
