## Audit complete — `config-audit` ConfigMap created in `demo` namespace

**14 findings across 4 severity levels:**

### HIGH
1. **Conflicting duplicate flags** — both `--watch-ingress-without-class=true` and `=false` are present in container args (config drift from a manual patch, generation=2). The last occurrence (`=false`) wins but the original intent was `=true`.
2. **`--publish-status-address=localhost`** — Ingress objects will advertise `localhost` as their external address, breaking external-dns and all tooling that reads Ingress status.
3. **No resource limits** — only requests set (`100m CPU`, `90Mi RAM`); the controller can consume unlimited node resources.
4. **`terminationGracePeriodSeconds: 0`** — immediate SIGKILL on pod stop; the preStop `/wait-shutdown` hook is never allowed to run; in-flight connections drop on every rollout.

### MEDIUM
5. **`readOnlyRootFilesystem: false`** — writable root filesystem increases attack surface if the process is compromised.
6. **hostPort 80/443** — bypasses NetworkPolicies; ties the pod to a single node.
7. **Tolerations for control-plane nodes** — controller can schedule on master/control-plane nodes.
8. **ClusterRole reads secrets cluster-wide** — controller can enumerate TLS secrets from every namespace.
9. **Server tokens not disabled** — ConfigMap is empty (`data: null`); nginx version disclosed in `Server` response header.
10. **`externalTrafficPolicy: Cluster`** — real client IPs are SNAT-ed to node IP before reaching the controller; breaks IP-based rate limiting and security logging.

### LOW
11. **Orphaned IngressClass resources** (`ingress-1`, `ingress-2`) with no matching controller — confirmed in logs.
12. **Demo app unreachable** — no Service and no Ingress object exist in the `demo` namespace; `demo-app` pod cannot receive traffic through the ingress.
13. **No pod security context** on `demo-app` and `curl-test` — run as root with no capability drops.

### INFO
14. **LoadBalancer IP unassigned** — expected in a kind cluster without MetalLB.

No changes were made to any existing cluster resource.
