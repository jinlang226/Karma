The upgrade is complete. Here's a summary of what was accomplished:

**Result:**
- All 3 nodes now run **RabbitMQ 4.1.8** on Erlang 27.3.4.13
- `/app` vhost still exists with `app-queue` intact (5 messages preserved, durable)
- Cluster is fully healthy — 3 disk nodes, no alarms, none under maintenance

**Upgrade path taken:** `3.9 → 3.11 → 3.12 → 3.13 → 4.0 → 4.1`

The key issues navigated:
1. `classic_mirrored_queue_version` was introduced in 3.10/3.11 but required by 3.12 — jumping 3.9→3.12 directly fails. A 3.11 intermediate hop was needed to surface and enable that flag.
2. `rabbitmqctl enable_feature_flag all` was run at each version before the next hop.
3. `queue_master_locator = min-masters` was removed from the ConfigMap before the 4.0 hop (that config key was dropped in 4.x).
