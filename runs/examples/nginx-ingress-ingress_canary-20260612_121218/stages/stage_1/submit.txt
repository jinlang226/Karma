Both routing paths work correctly:

- **Without `X-Canary` header** → `stable`
- **With `X-Canary: always`** → `canary`

**Root cause:** The `canary-canary` Ingress had `nginx.ingress.kubernetes.io/canary-by-header-value: "wrong"` instead of `"always"`, so the header check never matched. Fixed by patching the annotation to `"always"` (and updating the source file `resource/ingress-canary.yaml` to match).
