The 3-node CockroachDB cluster is fully deployed and healthy. Here's what was created:

**Resources deployed:**
- `crdb-cluster` — headless ClusterIP:None service (26257 gRPC, 8080 HTTP) with `publishNotReadyAddresses: true`
- `crdb-cluster-public` — public ClusterIP service (26257 gRPC, 8080 HTTP)
- `crdb-cluster` StatefulSet — 3/3 pods Ready, `cockroachdb/cockroach:v24.1.0`, `Parallel` pod management
- `crdb-cluster-pdb` — PodDisruptionBudget (`maxUnavailable: 1`) preserving quorum
- 3 × 10Gi PVCs all Bound

**Key decisions:**
- `podManagementPolicy: Parallel` was required — the default `OrderedReady` blocks pod 1 and 2 until pod 0 is Ready, but CockroachDB can't become Ready until `cockroach init` is run with all nodes present (chicken-and-egg).
- `publishNotReadyAddresses: true` on the headless service ensures DNS entries exist for joining nodes before they pass readiness checks.
- `cockroach init` was run once all 3 pods reached Running state to bootstrap the cluster.
