The command exits 0 and prints `pong`.

**Root cause:** `ray.init(address="ray-head:6379")` tries to join the cluster as a Ray node via the internal GCS port, which requires a local Ray session on disk (`node_ip_address.json`). An external client must use the Ray Client protocol instead: `ray://ray-head:10001`.

**Fix applied:** Updated `ray-job-script.yaml` ConfigMap to use `address="ray://ray-head:10001"` — the service already exposed port 10001, so no service change was needed.
