- **Cluster status: green**, 3 nodes, master elected (`es-cluster-0`, marked `*`)
- **Bootstrap setting removed**: `cluster.initial_master_nodes` is no longer in the ConfigMap
- **Rolling restart completed** — all 3 pods cycled one at a time, cluster stayed healthy throughout

**Root cause:** The `es-config` ConfigMap was missing `cluster.initial_master_nodes`, which Elasticsearch 7.x requires to bootstrap the initial quorum. Without it, no node could win an election and all three were stuck in "master not discovered yet".

**What was done:**
1. Added `cluster.initial_master_nodes: [es-cluster-0, es-cluster-1, es-cluster-2]` to the ConfigMap and restarted all pods simultaneously — cluster bootstrapped to green.
2. Removed `cluster.initial_master_nodes` from the ConfigMap — this is a bootstrap-only setting that must not persist, as it would prevent the cluster from reforming correctly after a full restart once the cluster state is already stored on disk.
3. Performed a rolling restart (one pod at a time) to bake in the clean config — cluster stayed green/yellow throughout, confirming the fix is safe for future restarts.
