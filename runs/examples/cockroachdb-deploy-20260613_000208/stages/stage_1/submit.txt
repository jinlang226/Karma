The CockroachDB cluster is fully deployed and healthy:

**Resources created:**
- **Headless Service** `crdb-cluster` (ClusterIP: None, grpc/http, `publishNotReadyAddresses: true` so pods can resolve each other's DNS before cluster init)
- **Public Service** `crdb-cluster-public` (ClusterIP: 10.96.108.60, grpc/http)
- **StatefulSet** `crdb-cluster` — 3/3 pods `1/1 Running`, `podManagementPolicy: Parallel` (required so all nodes start simultaneously for cluster init)
- **PVCs** — 3 × 10Gi bound volumes, one per pod
- **PodDisruptionBudget** `crdb-cluster-pdb` — `minAvailable: 2`, allowing at most 1 voluntary disruption (quorum preserved)

**Key design decisions:**
- `podManagementPolicy: Parallel` — necessary because CockroachDB nodes wait for `cockroach init` before becoming Ready; the default ordered policy would stall at pod-0 indefinitely
- `publishNotReadyAddresses: true` on the headless service — allows pod DNS FQDNs to resolve for the `--join` flag and `cockroach init` before pods pass their readiness probe
- `--insecure` mode with no TLS as specified; all nodes advertise via their pod FQDN (`$(POD_NAME).crdb-cluster.cockroachdb.svc.cluster.local`)
