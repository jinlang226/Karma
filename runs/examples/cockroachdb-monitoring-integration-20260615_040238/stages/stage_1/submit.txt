Both conditions verified. Here's what was done:

**Created a `ServiceMonitor`** in the `cockroachdb` namespace:

```yaml
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: cockroachdb
  namespace: cockroachdb
spec:
  selector:
    matchLabels:
      app.kubernetes.io/name: cockroachdb
      app.kubernetes.io/component: database
  endpoints:
  - port: http
    path: /_status/vars
    interval: 15s
    scrapeTimeout: 10s
```

**Why this works:**
- The `Prometheus` CR in `monitoring` has `serviceMonitorNamespaceSelector: matchNames: [cockroachdb]` — it picks up ServiceMonitors from the `cockroachdb` namespace
- The selector matches the `crdb-cluster` headless service (ClusterIP: None), so Prometheus discovers each pod's IP individually → scrape URLs resolve to pod IPs (what the oracle checks)
- The `http` port name on the service points to 8080, with path `/_status/vars`

**Result:** 3 active targets (`health=up`) — one per CockroachDB pod — and the metrics endpoint (`crdb-cluster-public:8080/_status/vars`) is reachable and returning `sys_uptime`.
