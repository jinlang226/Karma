The MongoDB replica set has been successfully scaled from 3 to 5 members:

- **replica-0**: PRIMARY (unchanged)
- **replica-1**: SECONDARY (unchanged)
- **replica-2**: SECONDARY (unchanged)
- **replica-3**: SECONDARY (new)
- **replica-4**: SECONDARY (new)

**What was done:**
1. `kubectl scale statefulset mongodb-replica --replicas=5` — provisioned 2 new pods with persistent volumes
2. `rs.add()` on the primary for each new member — registered them in the replica set config
3. New members completed `STARTUP2` (initial sync) and became `SECONDARY` within ~20 seconds

No data loss and no availability interruption — the primary stayed up throughout, and availability was maintained with the original 3 members while the new nodes synced.
