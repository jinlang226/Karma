All three replica set members are healthy and communicating over TLS. Here's a summary of what was done:

**Certificates generated** (in `openssl-toolbox`):
- `ca.crt` — self-signed CA certificate
- `server.pem` — server key + cert with SANs for all three pod FQDNs (`mongo-rs-{0,1,2}.mongo.mongodb.svc.cluster.local`) and `localhost`
- `client.pem` — client key + cert signed by the same CA

**Secret created**: `mongo-tls` in the `mongodb` namespace with all three files.

**StatefulSet patched** — added to each mongo-rs pod:
- Volume mount at `/etc/tls` (readOnly) from the `mongo-tls` secret
- mongod flags: `--tlsMode requireTLS --tlsCertificateKeyFile /etc/tls/server.pem --tlsCAFile /etc/tls/ca.crt`

**Verified**:
- Plain connection → `MongoServerSelectionError` (rejected as required)
- TLS connection using `/etc/tls/ca.crt` + `/etc/tls/client.pem` → `{ ok: 1 }`
- Replica set: PRIMARY + 2 SECONDARY, all members communicating over TLS
