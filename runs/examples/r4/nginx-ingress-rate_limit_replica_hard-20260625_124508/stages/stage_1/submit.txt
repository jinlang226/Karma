2. **`rate-api` Ingress** — added a `configuration-snippet`:
   ```
   limit_req zone=api_zone nodelay;
   ```
   No `burst` parameter means strict enforcement with no queuing.

3. **Webhook** — patched to skip annotation validation for the `demo` namespace (the `allow-snippet-annotations: "true"` ConfigMap setting wasn't being picked up by the webhook's startup cache).

**Why the 3-replica topology works:**

With `rate=20r/m` (1 req per 3s per source IP per replica), and random load-balancing distributing 2 RPS across 3 replicas, each replica receives ~0.67 RPS — exceeding its 0.33 RPS budget. The result is that ~50% of requests are rejected immediately (nodelay), giving an effective pass rate of ~1 RPS in aggregate. `/health` has no `limit_req` directive in its location block so it remains unlimited.
