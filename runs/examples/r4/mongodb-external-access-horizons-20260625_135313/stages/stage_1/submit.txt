**What was configured:**

| Member | Internal host | External horizon (`external`) |
|--------|--------------|-------------------------------|
| 0 | `mongo-rs-0.mongo.mongodb.svc.cluster.local:27017` | `domain-rs-1:31181` |

| 2 | `mongo-rs-2.mongo.mongodb.svc.cluster.local:27017` | `domain-rs-3:31183` |

**How split-horizon works:** When an external client connects to a member using an SNI hostname that matches a `horizons` key (requires TLS), MongoDB returns the addresses from that horizon in its `hello`/`isMaster` response — so the client's connection string stays entirely on external endpoints. Internal cluster clients connecting via the cluster-local DNS see the internal addresses (the default horizon). No NodePort services were modified.
