All 3 nodes are healthy and SQL is accepting connections. Here's the full summary:

---

**Root cause:** The StatefulSet command hardcoded `crdb-cluster.default.svc.cluster.local` in both `--advertise-host` and `--join`, but the pods and headless service live in the `cockroachdb` namespace. DNS lookups for `.default.svc.cluster.local` returned `no such host`.

**Fix applied:**
1. **Patched the StatefulSet** — changed the namespace in `--advertise-host` and `--join` from `.default.svc.cluster.local` → `.cockroachdb.svc.cluster.local`
2. **Force-restarted all 3 pods** simultaneously (using `kubectl delete pod`) so they picked up the new spec
3. **Ran `cockroach init`** via the direct cluster kubeconfig (the KARMA proxy kubeconfig doesn't support SPDY exec upgrades)

**Result:**
- All 3 pods `1/1 Running`
- `cockroach node status` shows nodes 1/2/3 — `is_available=true`, `is_live=true`
- SQL connections accepted on `crdb-cluster-public:26257`
