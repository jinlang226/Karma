Everything is working correctly:

- **HTTPS** (`https://es.example.com`) → returns Elasticsearch API response ✓
- **Plain HTTP** (`http://es.example.com`) → returns `308 Permanent Redirect` to HTTPS, not ES data ✓

**What was done:**

1. Generated a self-signed TLS certificate for `es.example.com` with openssl.
2. Created a `es-tls-secret` TLS secret in the `elasticsearch` namespace.
3. Created an `Ingress` resource (`es-ingress`) in the `elasticsearch` namespace with:
   - `ingressClassName: nginx` (uses the existing nginx ingress class)
   - TLS termination using `es-tls-secret`
   - `nginx.ingress.kubernetes.io/ssl-redirect: "true"` and `force-ssl-redirect: "true"` annotations to block plain HTTP access (returns 308)
   - Backend routed to `es-http:9200` (the existing ClusterIP service)
