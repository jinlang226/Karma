All pods are Ready and the replica set is healthy: **1 PRIMARY + 2 SECONDARY**.

**Root cause:** The `health-overrides` ConfigMap contained a stale password (`legacyHealthPass123`) for `mongodb-replica-1`, but MongoDB's `health-user` was created with `healthpass123` (from the secret) and that credential was replicated to all members. The health check on pod-1 used the override → SCRAM auth failed → readiness probe failed.

**Fix:** Removed the `mongodb-replica-1` key from the `health-overrides` ConfigMap so the probe falls back to the correct password from the `health-user-password` secret.
