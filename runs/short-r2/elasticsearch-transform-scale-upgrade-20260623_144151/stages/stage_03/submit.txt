Completed scale-up of the Elasticsearch cluster with two new data nodes.

Actions taken:
1. Created headless Service "es-cluster" (clusterIP: None, selector: app=es-cluster) so the es-cluster StatefulSet pods could resolve each other via DNS and form a cluster.
2. Patched the "es-http" ClusterIP Service selector from app=elasticsearch to app=es-cluster so it routes to the correct cluster.
3. Created ConfigMap "es-data-config" with node.attr.tier=warm and node.roles=[data,ingest].
4. Created StatefulSet "es-data" (2 replicas, image 7.17.9, label app=es-cluster, serviceName=es-cluster) — the pods joined the es-cluster cluster as warm-tier data nodes.
5. Created index "app-data" (3 primary shards, 1 replica).
6. Set index.routing.allocation.require.tier=warm on app-data — all 6 shard copies relocated to es-data-0 and es-data-1.

Final state:
