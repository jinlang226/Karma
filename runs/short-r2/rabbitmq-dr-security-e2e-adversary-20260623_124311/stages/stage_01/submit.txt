All goals met:

**Root cause:** `app-user` was missing the `configure` permission on the `/app` vhost. RabbitMQ requires configure permission to declare (create) queues. The user had write+read (`.*`) but configure was empty (`""`), so the HTTP API PUT to `/api/queues/%2Fapp/app-queue` returned `Access refused`.

**Fix applied:**
```
rabbitmqctl set_permissions -p /app app-user ".*" ".*" ".*"
```
