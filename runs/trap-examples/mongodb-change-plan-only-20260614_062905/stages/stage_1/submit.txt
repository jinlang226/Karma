The `change-plan` ConfigMap is created with the `plan.md` key. Here's a summary of what was found and what the plan covers:

**Current state inspected:**
- 3-member replica set (`mongodb-replica-{0,1,2}`), all data-bearing, equal priority/votes
- Non-default settings: `systemLog.verbosity: 1`, `slowOpThresholdMs: 200`, `journalCompressor: zlib`, auth enabled with keyFile
- Single user `admin-user` holding `userAdminAnyDatabase + clusterAdmin + readWriteAnyDatabase`
- Both SCRAM-SHA-1 and SCRAM-SHA-256 active

**8 gaps identified vs. target hardened baseline (plan stored in ConfigMap, no changes applied):**

| # | Gap | Impact |
|---|-----|--------|
| 1 | Write concern `w:1` → raise to `w:majority, wtimeout:5000` | Data loss on primary failover |
| 2 | SCRAM-SHA-1 still enabled → disable, SHA-256 only | Weaker auth negotiable |
| 3 | `admin-user` holds 3 combined roles → split into 3 purpose-scoped accounts | Least-privilege violation |
| 4 | No monitoring user → add `monitor-user` with `clusterMonitor` | Ops tools require admin today |
| 5 | No TLS → enable `requireTLS` with certificates | Plaintext client + replication traffic |
| 6 | `bindIpAll: true` → restrict to loopback + pod IP | Unnecessary attack surface |
| 7 | `slowOpThresholdMs: 200` → lower to `100` | Ops between 100–200 ms not profiled |
| 8 | No audit log → enable audit for auth/DDL events | No record of privilege use |
