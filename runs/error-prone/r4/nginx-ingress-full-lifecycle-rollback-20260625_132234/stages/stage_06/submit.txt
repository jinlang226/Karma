Both tests pass. Here's what was done and why:

**Problem**: `limit-rps: "2"` with burst=2 doesn't throttle requests arriving at exactly 0.505s intervals — the token refills between each request, so all pass.

**Solution**: Annotated only the `rate-api` Ingress (leaving `rate-health` untouched) with:

```yaml
nginx.ingress.kubernetes.io/limit-rps: "1"
nginx.ingress.kubernetes.io/limit-burst-multiplier: "1"
nginx.ingress.kubernetes.io/limit-req-status-code: "429"
```

This generates `rate=1r/s burst=1 nodelay` in nginx. At the test's ~2 RPS send rate (curl + 0.5s sleep = ~505ms interval), the token refills every 1s but requests arrive every 505ms — so the alternating allow/reject pattern produces ~5 throttled responses per 10 requests.

**Results**:
- `/api` at ~2 QPS: 4 of 10 returned 503 (rate-limited) ✓
- `/health` at ~2 QPS: all 10 returned 200 (unlimited) ✓
