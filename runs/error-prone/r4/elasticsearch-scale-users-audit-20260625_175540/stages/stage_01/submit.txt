The 3-node Elasticsearch cluster is fully deployed and operational. Here's what was deployed and the two issues encountered:

**Resources created:**
- `ConfigMap/es-config` — elasticsearch.yml with TLS + security config
- `Service/es-headless` (headless, `publishNotReadyAddresses: true`) — transport discovery
- `Service/es-http` (ClusterIP:9200) — HTTPS client access
- `PodDisruptionBudget/es-pdb` (minAvailable: 2)
- `StatefulSet/es` (3 replicas, `podManagementPolicy: Parallel`, 5Gi PVCs each)

**Two bootstrap issues solved:**
1. **Deadlock with OrderedReady**: StatefulSet waits for es-0 Ready before starting es-1/es-2, but the readiness probe checks `_cluster/health` which requires a quorum — fixed with `podManagementPolicy: Parallel` so all 3 start simultaneously, plus `timeoutSeconds: 10` on the probe.
2. **DNS NXDOMAIN for headless service**: Kubernetes only publishes endpoints for Ready pods by default; unready pods can't discover each other — fixed with `publishNotReadyAddresses: true` on the headless service.

**Verified:** cluster is `green` with 3 nodes, document write and read through `es-http` succeed.
