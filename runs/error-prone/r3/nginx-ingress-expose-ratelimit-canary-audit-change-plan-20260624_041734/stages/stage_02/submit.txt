**What was done:**

  - `nginx.ingress.kubernetes.io/limit-rps: "1"` — 1 request/second per client IP
  - `nginx.ingress.kubernetes.io/limit-burst-multiplier: "1"` — burst=1, so 2 tokens total before rejecting

The nginx ingress controller generated `limit_req zone=... rate=1r/s burst=1 nodelay` for the `/api` location. With requests arriving every ~500ms, the token bucket empties faster than it refills, causing roughly every other excess request to be rejected with 503. The `/health` Ingress was left untouched and remains unlimited.
