HTTPS is restored. The curl returned exit code 0 and printed "hello".

**Root cause:** The `demo-ingress` TLS secret `expired-tls-secret` contained a certificate with a validity window of Jan 1–2 2020 (deliberately expired). nginx was serving this cert, causing all TLS clients to reject it with certificate validation errors.

**Fix:** Generated a new leaf certificate for `demo.example.com` signed by the same Test CA (key at `/tmp/test-ca.key`), valid through June 2028, and replaced the secret's data. nginx picked up the new cert immediately, and the curl test now completes successfully with `SSL certificate verify ok` and returns `hello`.
