A 3-node CockroachDB cluster is running in cockroachdb.

Tables are organized by tenant using schemas:
- tenant_a.* belongs to Tenant A
- tenant_b.* belongs to Tenant B

Your task:
Apply a zone configuration to all tenant_b tables only with:
- num_replicas = 3
- gc.ttlseconds = 14400
- range_min_bytes = 134217728
- range_max_bytes = 536870912

Do not modify zone configs for tenant_a tables.

Useful commands:
```bash
kubectl -n cockroachdb exec crdb-cluster-0 -- ./cockroach sql --insecure --database=defaultdb -e "SHOW TABLES FROM tenant_b;"
```

Verification:
1. tenant_b tables have the required zone config:
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach sql --insecure --database=defaultdb -e "SHOW ZONE CONFIGURATION FOR TABLE tenant_b.<table>;"

2. tenant_a tables have no custom zone config:
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach sql --insecure --database=defaultdb -e "SELECT target FROM crdb_internal.zones WHERE target LIKE 'TABLE %tenant_a.%';"
