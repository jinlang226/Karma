HTTPS is fully working. The curl returns exit code 0 and prints "hello".

Here's a summary of what was broken and what I fixed:

**Root Cause 1: Expired TLS certificate**
The `expired-tls-secret` in the `demo` namespace contained a certificate that expired on January 2, 2020. I generated a new RSA certificate for `demo.example.com` signed by the existing test CA (`/tmp/test-ca.crt`/`.key`), valid until June 2028, and replaced the secret.

**Root Cause 2: Misconfigured ingress-nginx controller**
The `ingress-nginx-controller` deployment had garbled args — it had the correct `nginx` class args followed by an extra set of `ingress-nginx-1` args (`--controller-class=k8s.io/ingress-nginx-1 --ingress-class=ingress-1`). The last value wins in pflag, so the controller was running as the `ingress-1` controller, silently ignoring all `nginx` class ingresses including `demo-ingress`. I patched the deployment to remove the extra args, which triggered a rollout and caused the controller to properly recognize the `nginx` IngressClass and load the `demo-ingress` TLS configuration.
