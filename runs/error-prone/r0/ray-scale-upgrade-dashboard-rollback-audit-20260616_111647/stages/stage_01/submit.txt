Everything is working. Here's a summary of what was done:

**Deployed:**
- `ray-head` Deployment (1 replica) — `rayproject/ray:2.9.0`, starts GCS on port 6379
- `ray-head` Service — routes port 6379 to the head pod
- `ray-worker` Deployment (2 replicas) — connects to `ray-head:6379`

**Issue encountered:** In Ray 2.9.0, `ray.init(address='ray-head:6379')` (GCS/driver mode) requires a local raylet already running on the calling pod. The ray-client pod's `sleep infinity` command doesn't start one, so I ran `ray start --address=ray-head:6379 --num-cpus=0` directly in the pod to join the cluster as a node. The raylet daemon (PID 147) persists inside the pod independently.
