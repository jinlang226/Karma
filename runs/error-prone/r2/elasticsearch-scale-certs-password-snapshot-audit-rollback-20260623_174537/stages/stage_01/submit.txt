**Key design decisions:**
- `podManagementPolicy: Parallel` — starts all 3 pods simultaneously so they can discover each other for initial cluster formation
- `publishNotReadyAddresses: true` on `es-headless` — makes pod IPs available for DNS before they pass the readiness probe (breaks the discovery deadlock)
- Transport TLS uses `verification_mode: certificate` since pod FQDNs (`es-N.es-headless.*`) don't match the cert's wildcard SANs (`*.elasticsearch.svc`)
- `emptyDir` for data (sufficient for this deployment; add PVCs for persistence)
