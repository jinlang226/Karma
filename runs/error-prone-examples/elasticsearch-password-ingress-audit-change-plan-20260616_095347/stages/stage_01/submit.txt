**Two non-obvious issues resolved during deployment:**
1. `podManagementPolicy: Parallel` — required so all 3 pods start simultaneously; sequential startup deadlocks because `cluster.initial_master_nodes` needs quorum before any node becomes ready
2. `publishNotReadyAddresses: true` on the headless service — required so Kubernetes publishes DNS endpoints for non-ready pods; without it, nodes can't resolve each other's hostnames, which blocks cluster formation
