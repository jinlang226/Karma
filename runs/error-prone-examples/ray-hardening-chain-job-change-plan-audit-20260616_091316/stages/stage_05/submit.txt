The job exits 0 and prints `pong`. 

**What was done:**
1. **Root cause**: The original script used `ray://ray-head:10001` (Ray Client protocol) which timed out because port 10001 isn't exposed by the `ray-head` service.
2. **Fix**: Updated `ray-job-script` ConfigMap so `job.py` runs `ray start --address=ray-head:6379` first (joins the cluster as a worker node, creating the local session state), then calls `ray.init(address="ray-head:6379")` which now finds the local session and connects successfully.
3. **Deployment**: Recreated the `ray-client` pod (necessary because `subPath` ConfigMap mounts don't live-update) to pick up the new script.
