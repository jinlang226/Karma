A CockroachDB cluster is running v23.2 binaries and is ready for a major upgrade.

Your task:
Upgrade the cluster to v24.1.0 and finalize the major upgrade.

Requirements:
- Upgrade binaries first, then finalize the major upgrade.
- Keep the cluster available throughout.

Useful commands:
```bash
kubectl -n cockroachdb get pods
kubectl -n cockroachdb exec crdb-cluster-0 -- ./cockroach sql --insecure -e "SHOW CLUSTER SETTING version;"
```

Verification:
1. All pods on v24.1.0:
   kubectl -n cockroachdb get pods -o jsonpath='{range .items[*]}{.metadata.name}: {.spec.containers[0].image}{"\n"}{end}'

2. Cluster version finalized:
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach sql --insecure -e "SHOW CLUSTER SETTING version;"
   (Should show '24.1' not '23.2')

3. Preserve downgrade option cleared:
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach sql --insecure -e "SHOW CLUSTER SETTING cluster.preserve_downgrade_option;"
   (Should show empty)

4. Cluster healthy after finalization:
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach node status --insecure

5. Can use v24.1 features:
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach sql --insecure -e "SELECT version();"
