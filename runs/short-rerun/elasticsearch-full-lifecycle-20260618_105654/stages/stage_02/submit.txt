Deployed a 3-node Elasticsearch 8.11.1 cluster in the `elasticsearch` namespace
using only native Kubernetes resources (no CRDs/operators).

Resources created:
- ConfigMap es-config: elasticsearch.yml with security + HTTP/transport TLS enabled.
- Secret elastic-password (key "password") and Secret es-http-tls (tls.crt, tls.key,
  ca.crt) — these were missing in the actual cluster state, so they were created.
- Headless Service es-cluster (port 9300, publishNotReadyAddresses: true for bootstrap
  discovery) and Service es-http (HTTPS port 9200).
- StatefulSet es-cluster: 3 replicas, image
  docker.elastic.co/elasticsearch/elasticsearch:8.11.1, TLS certs mounted, ELASTIC_PASSWORD
  from the secret, exec readiness probe hitting https://localhost:9200/_cluster/health.
- PodDisruptionBudget es-cluster (minAvailable: 2).

Note: a pre-existing leftover deploy (image 7.17.9, no TLS/security) was torn down first.

Verification:
- All 3 nodes Ready; cluster health = green, number_of_nodes = 3, all on 8.11.1.
- Reachable over HTTPS on port 9200 through Service es-http.
- Write (POST karma-test/_doc/1) and read (GET karma-test/_doc/1) both succeeded.
