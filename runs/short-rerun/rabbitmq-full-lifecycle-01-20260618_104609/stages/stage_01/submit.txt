## Summary

**Root cause:** The `app-producer` was CrashLooping with `not_authorised / Access refused` because the `app-user` had an **empty `configure` permission** on the `/app` vhost. It had `write` and `read` (`.*`) but no `configure` regex, so it could not *declare* (create) `app-queue` — and the producer's first action is a `PUT .../queues/.../app-queue`.

**Fix:** Granted configure permission via the persistent cluster-wide command:
```
rabbitmqctl set_permissions -p /app app-user ".*" ".*" ".*"
```
Then restarted the producer deployment so it retried cleanly.

**Verification — all goals met:**
- ✅ `app-producer` is `1/1 Running`, 0 restarts, no errors in logs (publish loop running).
- ✅ `/app` vhost contains `app-queue`, type `classic`, with messages (8 and growing).
- ✅ Cluster healthy with all 3 nodes running (`rabbitmq-0/1/2`).

No CRDs/operators were used, and no data was lost (only a permission grant + producer restart).
