A Spark cluster and data skew benchmark job have been deployed, but there are
configuration issues that need to be fixed before the jobs can run successfully.

YOUR TASK:
1. Diagnose and fix the configuration bugs in the deployment
2. Apply optimization strategies to fix the data skew problem
3. Compare performance across different strategies

PROBLEM DESCRIPTION:
- Dataset: orders (large), products (small), customers (medium)
- Join on product_id has severe data skew
- 5 hot products (product_id 1-5) receive 50% of all orders
- This causes some partitions to be 10x larger than others
- Result: Slow query performance due to stragglers

STEP 1: DIAGNOSE AND FIX CLUSTER/JOB ISSUES
The initial deployment has bugs that prevent jobs from running.
Check the status and logs to identify and fix the issues.

```bash
# Check cluster and job status
kubectl get pods -n spark-skew
kubectl get jobs -n spark-skew

# Check Spark master logs for startup issues
kubectl logs -n spark-skew -l app=spark-master

# Check baseline job logs for errors
kubectl logs -n spark-skew -l strategy=baseline
```

STEP 2: ONCE FIXED - OBSERVE THE BASELINE
```bash
# Check baseline job status
kubectl get job spark-skew-baseline -n spark-skew

# Watch baseline job logs - observe the skew ratio and slow performance
kubectl logs -n spark-skew -l strategy=baseline -f
```

STEP 3: APPLY BROADCAST JOIN OPTIMIZATION
```bash
# Apply the broadcast join job (may need fixes too)
kubectl apply -f cases/spark/spark_data_skew/resource/skew-job-broadcast.yaml

# Watch the broadcast job - should be faster!
kubectl logs -n spark-skew -l strategy=broadcast -f
```

STEP 4: APPLY AQE OPTIMIZATION
```bash
# Apply the AQE (Adaptive Query Execution) job
kubectl apply -f cases/spark/spark_data_skew/resource/skew-job-aqe.yaml

# Watch the AQE job - Spark auto-handles skew
kubectl logs -n spark-skew -l strategy=aqe -f
```

STEP 5: COMPARE RESULTS
```bash
# Check all jobs completed
kubectl get jobs -n spark-skew

# View all job logs to compare performance
kubectl logs -n spark-skew -l strategy=baseline --tail=20
kubectl logs -n spark-skew -l strategy=broadcast --tail=20
kubectl logs -n spark-skew -l strategy=aqe --tail=20
```

SUCCESS CRITERIA:
- All configuration bugs fixed
- Baseline job completed successfully (shows slow performance)
- At least ONE optimization strategy applied (broadcast or aqe)
- Optimization shows performance improvement vs baseline
- All applied jobs completed successfully
