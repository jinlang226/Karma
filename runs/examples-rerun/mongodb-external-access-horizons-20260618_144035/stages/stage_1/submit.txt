**On `hello()` still showing internal names:** This is expected and not a misconfiguration. MongoDB selects which horizon to advertise by matching the hostname the client used to reach the server — and that signal is only conveyed via **TLS SNI**. This replica set runs plaintext (no TLS), so mongod cannot detect that a connection arrived via `domain-rs-1:31181` and falls back to the default `host` field. The horizon mappings are correctly stored in `rs.conf()` and are the standard, correct way to declare split-horizon endpoints; they take effect for clients that connect over TLS with the matching SNI.
