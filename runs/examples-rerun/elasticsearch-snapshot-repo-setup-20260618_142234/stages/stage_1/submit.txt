## What was wrong and what I did

**Diagnosis:**
- The S3 credentials live in the Secret `es-secure-settings`, mounted at `/mnt/secure-settings` and already loaded into each node's Elasticsearch **keystore** (`s3.client.default.access_key` / `s3.client.default.secret_key`) — the correct, secure place (no creds in `elasticsearch.yml` or env vars).
- The StatefulSet's init container tries to install `repository-s3` as a plugin, but in ES 8.11 **repository-s3 is a built-in module**, so the plugins dir is empty and that's expected — the S3 repository type was available all along.

**Actions:**
1. `POST _nodes/reload_secure_settings` — applied the keystore S3 credentials to all 3 running nodes (S3 client credentials are reloadable, so **no pod restart was needed**).
2. Registered repository **`minio-repo`** (type `s3`) pointing at `minio.elasticsearch.svc.cluster.local:9000`, bucket `es-backups`, with `protocol: http` and `path_style_access: true` (required for MinIO).
