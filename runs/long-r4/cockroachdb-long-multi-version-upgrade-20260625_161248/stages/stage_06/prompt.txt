A 3-node CockroachDB cluster is running an older minor version.

Your task:
Upgrade the cluster to v23.2.10 with a controlled rolling update.

Requirements:
- Keep the cluster available during the upgrade.
- Ensure the rollout fully completes.

Useful commands:
```bash
kubectl -n cockroachdb get pods
kubectl -n cockroachdb get sts crdb-cluster -o jsonpath='{.spec.updateStrategy.rollingUpdate.partition}'
```

Verification:
1. All pods running new version:
   kubectl -n cockroachdb get pods -o jsonpath='{range .items[*]}{.metadata.name}: {.spec.containers[0].image}{"\n"}{end}'
   (All should show v23.2.10)

2. StatefulSet uses new image:
   kubectl -n cockroachdb get sts crdb-cluster -o jsonpath='{.spec.template.spec.containers[0].image}'
   (Should be cockroachdb/cockroach:v23.2.10)

3. Verify version via SQL:
   kubectl -n cockroachdb exec crdb-cluster-0 -- ./cockroach sql --insecure -e "SELECT version();"
   (Should show v23.2.10)

4. All pods ready:
   kubectl -n cockroachdb get pods
   (All 3 pods should be Running and Ready 1/1)

5. Cluster health:
   kubectl -n cockroachdb exec crdb-cluster-0 -- ./cockroach node status --insecure
   (All 3 nodes should be healthy)

6. No partition set:
   kubectl -n cockroachdb get sts crdb-cluster -o jsonpath='{.spec.updateStrategy.rollingUpdate.partition}'
   (Should be 0 or empty)
