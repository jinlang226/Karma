TLS is fully enabled. Here's a summary of what was done:

**Certificates generated** (in `openssl-toolbox`) and stored in a `mongo-tls` Secret:
- `ca.crt` — self-signed CA
- `server.pem` — server cert+key with SANs covering `IP:127.0.0.1`, `localhost`, and all three pod FQDNs (`mongo-rs-{0,1,2}.mongo.mongodb.svc.cluster.local`)
- `client.pem` — client cert+key signed by the same CA

**StatefulSet patched** to:
- Mount the `mongo-tls` secret at `/etc/tls` on all pods
- Add `--tlsMode requireTLS --tlsCertificateKeyFile /etc/tls/server.pem --tlsCAFile /etc/tls/ca.crt` to the `mongod` command

**Verification results:**
- Plain `mongosh` connection → `MongoServerSelectionError: connection closed` (blocked)
- TLS `mongosh --tls --tlsCertificateKeyFile /etc/tls/client.pem --tlsCAFile /etc/tls/ca.crt` → `ok: 1` (succeeds)
- Replica set healthy: all 3 members online (PRIMARY + 2 SECONDARYs) communicating over TLS
