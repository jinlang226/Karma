The split-horizon configuration is working correctly. When connecting from the mongo-client via the external hostnames, the replica set now advertises the external endpoints (`domain-rs-1:31181`, `domain-rs-2:31182`, `domain-rs-3:31183`) instead of the internal cluster-local hostnames.

**Summary of what was done:**

- Used `rs.reconfig()` on the primary (`mongo-rs-2`) to add a `horizons` field to each replica set member config:
  - member 0 (`mongo-rs-0`): `external → domain-rs-1:31181`
  - member 1 (`mongo-rs-1`): `external → domain-rs-2:31182`
  - member 2 (`mongo-rs-2`): `external → domain-rs-3:31183`

Clients connecting via the external `domain-rs-*` hostnames now receive the external advertised addresses from `isMaster`/`hello`, while internal clients connecting via cluster-local DNS still see the internal pod hostnames. The NodePort services were not modified.
