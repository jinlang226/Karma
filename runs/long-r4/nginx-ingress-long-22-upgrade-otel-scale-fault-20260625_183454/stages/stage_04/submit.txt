Both routes work correctly:

- Without `X-Canary: always` → `stable`
- With `X-Canary: always` → `canary`

**Root cause:** Both `canary-stable` and `canary-canary` Ingresses declared `ingressClassName: nginx`, but the running controller was configured with class `ingress-1` (`k8s.io/ingress-nginx-1`). The controller silently ignored both, returning 404 for all traffic. Patching the `ingressClassName` to `ingress-1` on both Ingresses caused the controller to pick them up, reload NGINX, and properly apply the canary header routing.
