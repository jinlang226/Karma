Your team deployed a SparkPi application to the cluster, but it's not working.
The application should calculate Pi using Monte Carlo method and complete successfully.

Current status: The SparkPi Job exists but is not completing successfully.

Your task:
- Investigate why the job is failing
- Fix any issues you find
- Get the job to run successfully

KNOWN ISSUES TO FIX:
1. Wrong image name: contains "-nonexistent"
2. Wrong serviceAccountName: "spark-nonexistent" doesn't exist
3. Wrong memory format: "512" instead of "512Mi"
4. Missing RBAC permission: "pods" resource missing from Role

Diagnostic Commands:
```bash
# Check job status
kubectl get job spark-pi -n spark-pi

# Check pod status
kubectl get pods -n spark-pi

# Check job pod logs
kubectl logs -n spark-pi -l app=spark-pi

# Check events for errors
kubectl get events -n spark-pi --sort-by='.lastTimestamp'

# Check RBAC permissions
kubectl get role spark-pi-role -n spark-pi -o yaml
```

Fix Commands:
```bash
# Delete failed job first
kubectl delete job spark-pi -n spark-pi

# Fix the job YAML and re-apply:
# 1. Change image from "apache/spark:3.5.3-nonexistent" to "apache/spark:3.5.3"
# 2. Change serviceAccountName from "spark-nonexistent" to "spark-pi"
# 3. Change memory from "512" to "512Mi"

# Fix RBAC - add "pods" to resources:
kubectl patch role spark-pi-role -n spark-pi --type=json \
  -p='[{"op": "add", "path": "/rules/0/resources/-", "value": "pods"}]'

# Re-apply the fixed job
kubectl apply -f cases/spark/deploy_spark_pi/resource/spark-pi-job.yaml
```

Success criteria:
- SparkPi Job status shows "Completed"
- Pod logs contain the calculated Pi value
- All configuration issues are fixed
