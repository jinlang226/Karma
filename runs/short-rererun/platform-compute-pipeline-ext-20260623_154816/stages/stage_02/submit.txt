HTTP 200. The fix required two changes:

1. **Service patch** — added a `dashboard` port entry (`8265→8265`) to the ray-head Service so the name `ray-head:8265` resolves inside the cluster.

2. **Deployment patch** — changed the `ray start` command from `--include-dashboard=false` to `--include-dashboard=true --dashboard-host=0.0.0.0 --dashboard-port=8265`. The dashboard was already spawning (Ray bug/version quirk), but it was bound to `127.0.0.1`, making it unreachable via the pod's network interface. Binding to `0.0.0.0` lets the Service route traffic to it.
