**What was deployed:**
- **Headless service** (`crdb-cluster`) with `publishNotReadyAddresses: true` so DNS for all pods resolves even before Ready, enabling the join handshake.
- **Public service** (`crdb-cluster-public`) for client access on gRPC (26257) and HTTP admin UI (8080).
- **StatefulSet** with `podManagementPolicy: Parallel` (required because all 3 nodes must start simultaneously to form quorum), insecure mode, pod DNS advertise addresses, and a 10Gi PVC per node.
- **PodDisruptionBudget** (`minAvailable: 2`) ensuring voluntary disruptions leave at least 2 of 3 nodes up, preserving quorum.
- Cluster bootstrapped with `cockroach init` after all pods reached Running state.
