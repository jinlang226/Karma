Everything is working. Here's a summary of what was done and why it worked:

**Root causes found and fixed:**

1. **Missing `es-transform-config` ConfigMap** — The StatefulSet tried to mount it but it didn't exist, preventing the pod from starting. Created it with the proper Elasticsearch configuration.

2. **Version mismatch** — The `es-transform` StatefulSet was pinned to image `7.17.9` while the main cluster runs `8.11.1`. ES nodes must be the same major version to join a cluster. Updated the image to `8.11.1`.

3. **Missing TLS certs volume** — The pod had no mount for the `es-http-tls` secret, so it couldn't use the transport SSL required by the secured cluster. Patched the StatefulSet to add the certs volume and mount.

4. **Transform never created** — The precondition's setup commands all used HTTP against an HTTPS-only cluster (exit code 52), so the transform was never created and source data was never inserted. Set up the `app-events` index with sample data, created the `events-by-service` transform, and started it.

**Result:** `es-transform-0` joined the 4-node cluster with `drt` roles (data, remote_cluster_client, transform). The transform is in `started` state, checkpoint advanced to 1, and the `app-events-rollup` index contains the aggregated results (3 service groups from 4 events).
