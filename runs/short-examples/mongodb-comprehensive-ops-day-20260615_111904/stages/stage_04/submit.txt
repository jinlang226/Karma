TLS is fully enabled and the replica set is healthy. Here's what was done:

1. **Generated certificates** in the `openssl-toolbox` pod:
   - CA key + self-signed cert (`ca.crt`)
   - Server key + cert signed by CA, with SANs covering all pod DNS names → concatenated as `server.pem`
   - Client key + cert signed by CA → concatenated as `client.pem`

2. **Created Secret `mongo-tls`** in the `mongodb` namespace with all three files.

3. **Patched the StatefulSet** to:
   - Add `--tlsMode requireTLS`, `--tlsCertificateKeyFile`, `--tlsCAFile`, `--tlsClusterFile`, `--tlsClusterCAFile` to `mongod`
   - Mount the `mongo-tls` secret at `/etc/tls` (read-only, mode 0400) on all pods

4. **Verified**:
   - Plain `mongosh` → `MongoServerSelectionError` (connection rejected)
   - TLS `mongosh` with `/etc/tls/client.pem` + `/etc/tls/ca.crt` → `{ ok: 1 }`
   - All three replica-set members communicating over TLS (PRIMARY + 2 SECONDARY)
