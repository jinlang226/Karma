The job exits 0 and prints "pong". Here's what was done:

1. **Root cause**: The original script used `ray://ray-head:10001` (Ray Client protocol), which fails because port 10001 isn't exposed by the `ray-head` Service. Switching to `ray-head:6379` (GCS direct) failed too — Ray's direct GCS connection requires a local session directory (`/tmp/ray/session_.../node_ip_address.json`) that only exists after `ray start` runs on the node.

2. **Fix**: Updated the ConfigMap `ray-job-script` so the script first runs `ray start --address=ray-head:6379 --num-cpus=0` (joining the cluster as a zero-CPU node, creating the needed session files), then calls `ray.init(address="ray-head:6379")`. Since subPath mounts don't auto-refresh, the `ray-client` pod was recreated to pick up the new ConfigMap.
