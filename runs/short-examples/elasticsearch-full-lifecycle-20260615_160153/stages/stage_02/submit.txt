Deployed a 3-node Elasticsearch 8.11.1 cluster using native Kubernetes resources.

Steps taken:
1. Generated self-signed TLS certificates (CA + server cert with SANs covering all service and pod DNS names)
2. Created es-http-tls secret (tls.crt, tls.key, ca.crt) and elastic-password secret (password=elasticpass)
3. Deleted the pre-existing ES 7.17.9 StatefulSet and its PVCs (incompatible data format)
4. Updated ConfigMap (es-config) with ES 8.11.1 settings: xpack.security.enabled, HTTP TLS, and transport TLS (verification_mode=certificate)
5. Deployed new StatefulSet with podManagementPolicy=Parallel (so all 3 nodes start simultaneously and form quorum)
6. Fixed service selectors (es-cluster and es-http) to match pod label app=elasticsearch

Verification:
- All 3 pods Ready: kubectl get pods -n elasticsearch
- Cluster health: green, 3 nodes
- HTTPS access on es-http:9200 works with elastic:elasticpass credentials
- Write + read test document succeeded
