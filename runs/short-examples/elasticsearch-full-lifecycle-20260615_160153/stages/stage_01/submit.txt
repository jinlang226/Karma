Oracle passes. Here's what was done and why:

**Root cause:** The ConfigMap had `discovery.seed_hosts` (peers to connect to) but was missing `cluster.initial_master_nodes`. In Elasticsearch 7.x, this setting is required for the very first master election on a fresh cluster — without it, every node waits forever for a master that can never be elected.

**Fix (two-phase):**
1. Added `cluster.initial_master_nodes: [es-cluster-0, es-cluster-1, es-cluster-2]` to the ConfigMap → rolling restart → cluster formed (green, 3 nodes, master elected).
2. Removed `cluster.initial_master_nodes` from the ConfigMap → rolling restart → cluster stayed green. This is the critical cleanup: leaving the setting in place is dangerous because on subsequent restarts it can prevent nodes from rejoining an already-formed cluster (it signals "ignore existing cluster state and form a new one").
