**Fix applied**:
1. Added `cluster.initial_master_nodes: [es-cluster-0, es-cluster-1, es-cluster-2]` to the ConfigMap and restarted all pods — cluster bootstrapped and elected a master.
2. Removed `cluster.initial_master_nodes` from the ConfigMap — this is intentional: after bootstrap the cluster state is persisted to the data PVCs, so the setting must not remain or it risks accidental re-bootstrapping on future restarts.
3. Rolling restart confirmed the cluster survives restarts with only `discovery.seed_hosts` (no bootstrap setting needed once the cluster is formed).
