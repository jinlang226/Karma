The task is complete. Here's what was done:

**Root cause:** The original script used `ray://ray-head:10001` (Ray Client protocol), but the `ray-head` Service only exposes port 6379 (GCS) and 8265 (dashboard) — port 10001 (Ray Client Server) was not exposed.

**Fix:** Updated the `ray-job-script` ConfigMap with a script that:
1. Calls `ray start --address=ray-head:6379` to register the client pod as a worker node in the Ray cluster
2. Waits briefly for registration
3. Calls `ray.init(address="ray-head:6379")` — this now works because the pod's IP is registered with the cluster
4. Prints `"pong"` and shuts down

The pod was recreated to pick up the ConfigMap change (required because `subPath` mounts don't live-update). The test `kubectl -n ray exec ray-client -- python /opt/job.py` now exits 0 and prints `pong`.
