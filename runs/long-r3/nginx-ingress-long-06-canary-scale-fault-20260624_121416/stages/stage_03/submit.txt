Both routes work correctly:

- Without header → `stable`
- With `X-Canary: beta` → `canary`

**Root cause:** The `canary-canary` Ingress had `canary-by-header-value: always` (a special nginx-ingress keyword that means "always send to canary regardless of header value") instead of the literal string `beta`. Patching it to `beta` makes nginx-ingress only activate the canary backend when the `X-Canary` header contains exactly `beta`.
