The split-horizon configuration is working correctly. When connecting via the external NodePort addresses, the replica set now advertises the external hostnames (`domain-rs-1:31181`, `domain-rs-2:31182`, `domain-rs-3:31183`) instead of the internal cluster-local names.

**What was done:**

The `horizons` field was added to each replica set member via `rs.reconfig()` on the primary (`mongo-rs-2`):

| Member | Internal host | External horizon |
|--------|--------------|-----------------|
| 0 | `mongo-rs-0.mongo.mongodb.svc.cluster.local:27017` | `domain-rs-1:31181` |
| 1 | `mongo-rs-1.mongo.mongodb.svc.cluster.local:27017` | `domain-rs-2:31182` |
| 2 | `mongo-rs-2.mongo.mongodb.svc.cluster.local:27017` | `domain-rs-3:31183` |

Clients connecting through the NodePort endpoints now receive the external hostnames in `hello`/`isMaster` responses, so drivers can route subsequent connections correctly without trying to reach the internal cluster-local addresses.
