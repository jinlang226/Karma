Everything is in place and verified. Here's the summary:

**What's configured:**

The `elasticsearch-https` Ingress in the `elasticsearch` namespace is configured with:
- **Ingress class:** `nginx`
- **Host:** `es.example.com` with TLS secret `es-ingress-tls`
- **Backend:** `es-http:9200`
- **Key annotations:**
  - `nginx.ingress.kubernetes.io/backend-protocol: HTTPS` — tells nginx to speak HTTPS to the ES backend (ES only listens on TLS)
  - `nginx.ingress.kubernetes.io/force-ssl-redirect: "true"` + `ssl-redirect: "true"` — HTTP is never passed through
  - `nginx.ingress.kubernetes.io/proxy-ssl-verify: "off"` — skips verification of ES's self-signed cert

**Verified behavior:**
- `https://es.example.com` → reaches Elasticsearch, returns `401` (auth required — ES is live and responding)
- `http://es.example.com` → `308 Permanent Redirect` to HTTPS, never touches ES

**Cluster health:** all 3 ES pods running, ingress-nginx controller running, no restarts.
