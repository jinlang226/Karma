**Root cause:** The probes had been modified to values far too aggressive for a MongoDB auth-enabled replica set:
- `timeoutSeconds: 1` — `mongosh` with keyfile auth reliably takes >1s to connect, so every probe timed out
- `initialDelaySeconds: 5/15` — too short for MongoDB to initialize before probes started firing
- `failureThreshold: 2` — only 2 failures before the pod was marked unhealthy/killed

**Fix:** Patched the StatefulSet to restore sensible values — `timeoutSeconds: 5`, `initialDelaySeconds: 30/60`, `failureThreshold: 6/10` — matching the original intended configuration from the `last-applied-configuration` annotation.
