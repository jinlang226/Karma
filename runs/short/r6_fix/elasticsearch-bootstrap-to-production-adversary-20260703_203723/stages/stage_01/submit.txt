All goals achieved. Here's what was done:

**Root cause:** The `es-config` ConfigMap was missing `cluster.initial_master_nodes`, which Elasticsearch 7.x requires to bootstrap a brand-new cluster. Without it, nodes could discover each other via `discovery.seed_hosts` but couldn't elect an initial master — hence "master not discovered yet".

**Fix applied:**
1. **Added `cluster.initial_master_nodes`** to the ConfigMap with all 3 node names, enabling the initial master election quorum.
2. **Restarted all 3 pods simultaneously** (deleted them; the StatefulSet recreated them) so they picked up the new config.
3. **Cluster formed** and reached `green` status with 3 nodes and `es-cluster-1` elected as master.
4. **Removed `cluster.initial_master_nodes`** from the ConfigMap — this is a bootstrap-only setting that is dangerous to keep (could cause split-brain on re-bootstrap after a full data wipe).
5. **Rolling restart** (`kubectl rollout restart statefulset/es-cluster`) confirmed the cluster stays green through the restart cycle, with each pod rejoining using the persisted cluster state rather than the bootstrap setting.

**Final state:** Cluster is `green`, 3 nodes (all `1/1 Running`), master elected (`es-cluster-1`), `cluster.initial_master_nodes` absent from the ConfigMap.
