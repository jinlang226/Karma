All goals met:

**Root cause:** The `es-config` ConfigMap was missing `cluster.initial_master_nodes`, which Elasticsearch 7.x requires to bootstrap a brand-new cluster (no existing cluster state). Without it, all nodes loop indefinitely on "master not discovered yet."

**What was done:**

1. **Added `cluster.initial_master_nodes`** (`es-cluster-0`, `es-cluster-1`, `es-cluster-2`) to the ConfigMap and restarted all 3 pods simultaneously — the cluster bootstrapped and reached **green** status with a master elected (`es-cluster-1`).

2. **Removed `cluster.initial_master_nodes`** from the ConfigMap immediately after cluster formation. This setting is bootstrap-only: once the cluster has a UUID and persisted state, it must not be present, as it would cause problems if a node were added later.

3. **Rolling restart verified** — each node was cycled one at a time. The cluster dropped to 2 nodes (yellow) during each removal, returned to green once the pod rejoined, and ended **green with 3 nodes and a master** — with no `cluster.initial_master_nodes` in the config.
