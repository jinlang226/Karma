All three pods are `1/1 Ready` and the replica set has the correct topology. Here's a summary of the changes made:

**Root causes fixed:**

1. **`timeoutSeconds: 1`** — mongosh takes >1s to start, causing every probe to time out. Increased to `10s`.

2. **`failureThreshold: 2`** — Too aggressive; pods were killed after just 2 consecutive failures. Increased to `6`.

3. **`initialDelaySeconds` too short** — MongoDB needs time to start up and join the replica set. Increased from `5s → 30s` (readiness) and `15s → 60s` (liveness).

4. **`systemLog.verbosity: 4`** — DEBUG-level logging caused heavy I/O overhead, contributing to slow command execution. Reduced to `0` (default INFO level).

**Result:**
- All 3 pods: `1/1 Running`, 0 restarts
- `mongodb-replica-1`: PRIMARY  
- `mongodb-replica-0`, `mongodb-replica-2`: SECONDARY
- Replica set name: `mongodb-replica`, all 3 hosts registered
