Everything is in order. Here's a summary of what was done:

**Problem:** The namespace had an ES 7.17.9 StatefulSet running without TLS/auth, missing the required secrets, and using sequential pod startup (causing a bootstrap deadlock for a 3-node cluster).

**Actions taken:**

1. **Generated TLS certificates** — CA + node cert with SANs covering all pod DNS names and the `es-http` service hostname.

2. **Created secrets** — `es-http-tls` (tls.crt, tls.key, ca.crt) and `elastic-password`.

3. **Deleted old 7.x StatefulSet and PVCs** — data format is incompatible with 8.x.

4. **Updated ConfigMap** for ES 8.x: added `cluster.initial_master_nodes`, `xpack.security.enabled: true`, and full HTTP+transport TLS configuration pointing to `/usr/share/elasticsearch/config/certs/`.

5. **Deployed new StatefulSet** with:
   - Image `docker.elastic.co/elasticsearch/elasticsearch:8.11.1`
   - `podManagementPolicy: Parallel` — all 3 pods start simultaneously, avoiding the quorum deadlock
   - `ELASTIC_PASSWORD` from the secret
   - TLS cert volume mounted at `/usr/share/elasticsearch/config/certs/`

**Verified:** Cluster status is **green** with 3 nodes, HTTPS write and read both succeed through the `es-http` service.
