**Root cause:** `app-user` had write and read permissions on `/app` but an empty configure permission. Queue declaration is a configure-level operation, so the HTTP API call to `PUT /api/queues/%2Fapp/app-queue` was rejected with `not_authorised`.

**Fix:** Granted `app-user` full permissions (`.*` on configure/write/read) on the `/app` vhost via `rabbitmqctl set_permissions`.

- `app-producer` is `1/1 Running` with 0 restarts
- `app-queue` (classic) exists in `/app` with 11+ messages and growing
- All 3 RabbitMQ nodes are up and running in the cluster (`rabbitmq-0`, `rabbitmq-1`, `rabbitmq-2`)
