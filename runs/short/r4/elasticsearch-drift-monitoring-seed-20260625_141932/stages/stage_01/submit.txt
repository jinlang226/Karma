Everything is working. Here's a summary of what was deployed:

**Resources created:**

| Resource | Name | Purpose |
|---|---|---|
| Service | `es-headless` | Headless service for pod DNS discovery (`publishNotReadyAddresses: true`) |
| Service | `es-http` | ClusterIP HTTPS access on port 9200 |
| ConfigMap | `es-config` | `elasticsearch.yml` with security + TLS config |
| StatefulSet | `es` | 3-node cluster, `podManagementPolicy: Parallel`, 5Gi PVCs each |

**Key configuration decisions:**
- `podManagementPolicy: Parallel` — avoids the chicken-and-egg deadlock where OrderedReady blocks es-1/2 until es-0 is Ready, but es-0 needs es-1/2 for quorum
- `publishNotReadyAddresses: true` on the headless service — makes DNS resolve for all pods immediately so nodes can discover each other before any are Ready
- Transport SSL reuses the `es-http-tls` secret with `verification_mode: certificate` (no hostname check needed between pods)

**Verified:** Cluster is green, 3/3 nodes, HTTPS reads and writes succeed via `es-http` service.
