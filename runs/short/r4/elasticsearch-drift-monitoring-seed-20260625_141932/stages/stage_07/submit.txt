Everything is complete. Here's a summary of what was done:

**Problem:** The `es-secure-settings` secret didn't exist, there was no MinIO service, the `repository-s3` plugin wasn't installed on any node, and the `es-backups` bucket didn't exist.

**Solution:**
1. **Created MinIO service** — `minio.elasticsearch.svc.cluster.local:9000` (the deployment existed but had no service)
2. **Created `es-secure-settings` secret** — stores `s3.client.default.access_key` and `s3.client.default.secret_key` (both `minioadmin`) as Kubernetes Secret data, not in elasticsearch.yml or env vars
3. **Patched both StatefulSets** (`es-cluster` and `es-data`) with an init container that:
   - Installs the `repository-s3` plugin and copies it to a shared `emptyDir` volume
   - Reads credentials from the secret and populates the Elasticsearch keystore, copying it to another shared `emptyDir` volume
   - The main container mounts these volumes so the plugin and keystore persist across container restarts (but not pod replacements — the init container re-runs on each pod start, which is correct)
4. **Created `es-backups` bucket** in MinIO via `mc` directly
5. **Registered `minio-repo`** as an S3 repository with `path_style_access: true` and `protocol: http`
6. **Created `snapshot-1`** — state `SUCCESS`, 3/3 shards across all 5 nodes
