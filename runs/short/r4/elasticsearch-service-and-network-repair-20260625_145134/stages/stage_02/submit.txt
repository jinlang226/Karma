All three verification conditions are met:

1. **Cluster health: green** — `status: green`, no unassigned shards.
2. **All 3 nodes in `/_cat/nodes`** — `es-cluster-0`, `es-cluster-1`, and `es-cluster-2` all appear.
3. **`app-data` readable** — search returns the seeded document.

**Root cause:** The precondition script applied a ConfigMap via a heredoc that shell-expanded `${POD_NAME}` to an empty string, leaving `node.name: ` blank. Since the ConfigMap uses `subPath` mounting, `es-cluster-2` (restarted after the broken ConfigMap was applied) read the empty `node.name` and crashed on every restart.

**Fix:** Restored the original ConfigMap (with literal `${POD_NAME}` for Elasticsearch's own env-var substitution and correct short-form seed hosts), then deleted `es-cluster-2` so the StatefulSet controller recreated it with the correct config.
