All three files (`ca.crt`, `server.pem`, `client.pem`) are mounted at `/etc/tls` in every mongo-rs pod.

**Summary of what was done:**

1. **Generated certificates** in the `openssl-toolbox` pod:
   - CA key + self-signed certificate (`ca.crt`)
   - Server key + certificate with SANs for all 3 replica set DNS names (`server.pem` = cert + key)
   - Client key + certificate (`client.pem` = cert + key)

2. **Created a Kubernetes secret** `mongo-tls` with the three PEM files.

3. **Patched the `mongo-rs` StatefulSet** to:
   - Add `--tlsMode requireTLS`, `--tlsCertificateKeyFile /etc/tls/server.pem`, `--tlsCAFile /etc/tls/ca.crt` to the `mongod` command
   - Mount the `mongo-tls` secret at `/etc/tls`

4. **Verified:**
   - Plain connection → **rejected** (`MongoServerSelectionError`)
   - TLS connection using `/etc/tls/ca.crt` + `/etc/tls/client.pem` → **succeeds** (`ok: 1`)
   - Replica set healthy: 3 members (1 primary, 2 secondaries) communicating over TLS
