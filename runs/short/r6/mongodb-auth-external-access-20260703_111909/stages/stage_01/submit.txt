The fix is complete. Here's what was done:

1. **Root cause**: `reporting-user` had only the `rawRead` role, which granted `find` on `appdb.raw` — the ingestion data that must not be exposed. It had no access to `appdb.reports` where dashboards read from.

2. **Fix**:
   - Created a new custom role `reportsRead` on `appdb` with `find` privilege scoped to `appdb.reports` only
   - Updated `reporting-user` to use `reportsRead` instead of `rawRead`

3. **Result**: `reporting-user` can now query `appdb.reports` (3 docs found) and is denied access to `appdb.raw` — minimum privilege, no raw data exposure.
