**What was done:**

1. **Root cause:** The precondition set up `reporting-user` with the `rawRead` role — which grants `find` on `appdb.raw` only. The user had access to the wrong collection (raw ingestion data) and no access to `appdb.reports`.

2. **Fix:**
   - Created a new custom role `reportsRead` on `appdb` with `find` privilege scoped exclusively to `appdb.reports`
   - Updated `reporting-user` to use only `reportsRead`, removing `rawRead`

The reporting dashboards now have the minimum required access (`find` on `appdb.reports`) without exposing `appdb.raw`.
