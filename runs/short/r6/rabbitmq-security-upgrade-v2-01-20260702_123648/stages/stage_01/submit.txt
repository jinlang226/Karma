Everything is healthy. Here's a summary of what was wrong and what was fixed:

**Root cause:** `app-user` was missing the `configure` permission on the `/app` vhost. The RabbitMQ Management API's `PUT /queues/{vhost}/{queue}` endpoint requires `configure` permission to declare a queue. The user had `write` and `read` but the `configure` field was blank.

**Fix applied:**
```
rabbitmqctl set_permissions -p /app app-user ".*" ".*" ".*"
```

**Verified end state:**
- All 3 RabbitMQ nodes (`rabbitmq-0/1/2`) are Running
- `app-producer` is `1/1 Running` with 0 restarts
- `/app` vhost contains `app-queue` of type `classic` (durable) with messages accumulating (29+ and growing)
- `app-user` now has full `configure/write/read` permissions on `/app`
