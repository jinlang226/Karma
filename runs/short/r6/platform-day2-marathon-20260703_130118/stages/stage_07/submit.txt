Everything is working. Here's a summary of what was deployed:

**Resources created:**

| Resource | Name | Purpose |
|---|---|---|
| ConfigMap | `es-config` | `elasticsearch.yml` with TLS + security config |
| Service (Headless) | `es-transport` | Pod-to-pod discovery on port 9300 (`publishNotReadyAddresses: true`) |
| Service (ClusterIP) | `es-http` | HTTPS access on port 9200 |
| PodDisruptionBudget | `es-pdb` | Ensures ≥2 nodes stay up during disruptions |
| StatefulSet | `es` | 3-node cluster, `podManagementPolicy: Parallel` |

**Key decisions:**
- `podManagementPolicy: Parallel` — all 3 pods start simultaneously; sequential startup causes a DNS deadlock since ES won't become Ready until the cluster forms, and the cluster can't form until all nodes can resolve each other
- `publishNotReadyAddresses: true` on `es-transport` — without this, CoreDNS only creates records for Ready pods, which also breaks bootstrap
- Transport TLS uses `verification_mode: certificate` since the existing cert's SANs (`*.elasticsearch.svc`) don't cover pod FQDNs
- Password set via `ELASTIC_PASSWORD` env var from the pre-existing `elastic-password` secret
