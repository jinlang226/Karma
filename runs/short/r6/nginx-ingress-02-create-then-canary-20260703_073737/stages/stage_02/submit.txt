Both routing rules work correctly:

- **Without header** → `stable`
- **With `X-Canary: always`** → `canary`

**Root cause:** The `canary-canary` ingress had annotation `nginx.ingress.kubernetes.io/canary-by-header-value: wrong` — the value the controller was matching against was the literal string `"wrong"` instead of `"always"`. A single `kubectl annotate --overwrite` fixed it.
