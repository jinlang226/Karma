**Root cause and fix:** The ConfigMap `es-config` had been corrupted with two bugs: `node.name` was set to empty (the `${POD_NAME}` shell variable got expanded to nothing when the ConfigMap was applied via a heredoc), and `discovery.seed_hosts` pointed to the wrong namespace (`default` instead of `elasticsearch`). es-cluster-0 and es-cluster-1 stayed up because the `subPath` mount type doesn't hot-reload ConfigMap changes. es-cluster-2 crashed on restart and couldn't rejoin because it picked up the broken config. Patching the ConfigMap with the correct `node.name: ${POD_NAME}` (literal, for Elasticsearch's own variable interpolation) and short-form seed hostnames (`es-cluster-N.es-cluster`), then deleting the Error pod so the StatefulSet recreated it, restored full cluster membership.
