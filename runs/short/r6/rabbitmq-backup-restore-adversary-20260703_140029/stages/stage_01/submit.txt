Everything is in order:

**Root cause:** `app-user` had write and read permissions on the `/app` vhost but the configure permission was empty (`""`). Declaring a queue requires configure access, which caused the `Access refused` error.

**Fix applied:**
```
rabbitmqctl set_permissions -p /app app-user ".*" ".*" ".*"
```

**Verified state:**
- `app-producer` — `1/1 Running`, 0 restarts
- `app-queue` in `/app` vhost — type `classic`, durable, already has 5 messages and counting
- All 3 RabbitMQ nodes (`rabbitmq-0/1/2`) — `1/1 Running`
