**Root cause:** `app-user` had an empty configure permission on the `/app` vhost (`"" .* .*`), so the RabbitMQ Management API rejected the queue-declare PUT with "Access refused."

**Fix:** Granted `app-user` full permissions on `/app`:
```
rabbitmqctl set_permissions -p /app app-user ".*" ".*" ".*"
```
