The CockroachDB cluster is now fully operational. Here's what was done:

**Root cause:** An adversary process had scaled the StatefulSet from 3 replicas down to 1, causing pods `crdb-cluster-1` and `crdb-cluster-2` to terminate. Pod 0 was waiting for the others and had self-initialized as a single-node cluster.

**Fix:** Scaled the StatefulSet back to 3 replicas:
```bash
kubectl -n cockroachdb scale statefulset crdb-cluster --replicas=3
```

**Result:**
- All 3 pods running (`crdb-cluster-0`, `crdb-cluster-1`, `crdb-cluster-2`)
- All 3 nodes healthy (`is_available: true`, `is_live: true`)
- SQL connections accepted — `SHOW DATABASES` returns system databases
- Node 0 had already initialized the cluster; nodes 1 and 2 joined via the `--join` flag pointing to all three headless service DNS names
