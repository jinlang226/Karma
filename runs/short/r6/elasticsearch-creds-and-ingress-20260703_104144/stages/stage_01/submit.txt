Everything is working. Here's a summary of what was deployed:

**Resources created:**

| Resource | Name | Purpose |
|---|---|---|
| ConfigMap | `elasticsearch-config` | `elasticsearch.yml` with TLS + discovery config |
| Service (headless) | `es-headless` | Pod-to-pod transport discovery (port 9300/9200, `publishNotReadyAddresses: true`) |
| Service (ClusterIP) | `es-http` | Client HTTPS access on port 9200 |
| PodDisruptionBudget | `elasticsearch-pdb` | `minAvailable: 2` |
| StatefulSet | `elasticsearch` | 3 replicas, `podManagementPolicy: Parallel` |

**Key design decisions:**
- `publishNotReadyAddresses: true` on the headless service so pods can resolve each other during bootstrap before any are Ready
- `podManagementPolicy: Parallel` so all 3 pods start simultaneously and reach quorum together (instead of OrderedReady, which deadlocks: pod-0 can't reach 2/3 quorum alone)
- TLS used for both HTTP and transport (`verification_mode: certificate` for transport to avoid hostname SAN issues)
- `ELASTIC_PASSWORD` injected from the existing `elastic-password` secret

**Verified:** cluster status green, 3 nodes, writes and reads succeed over HTTPS via `es-http.elasticsearch.svc.cluster.local:9200`.
