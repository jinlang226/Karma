Everything works. Here's what was configured:

**What was done:**

1. **Generated a self-signed TLS certificate** for `es.example.com` and stored it as the `es-example-tls` secret in the `elasticsearch` namespace.

2. **Created an Ingress** (`es-ingress`) in the `elasticsearch` namespace with:
   - `ingressClassName: nginx` — uses the nginx ingress class
   - TLS termination at `es.example.com` using the new cert
   - Routing `/` → `es-http:9200` (all 3 Elasticsearch pods)
   - `backend-protocol: HTTPS` — tells nginx to connect to the Elasticsearch backend over HTTPS (since ES 8.x uses TLS on port 9200)
   - `proxy-ssl-verify: off` — skips verification of the backend's self-signed cert (its SANs don't include `es.example.com`)
   - `force-ssl-redirect: true` + `ssl-redirect: true` — plain HTTP is rejected with a **308 Permanent Redirect** to HTTPS

**Test results:**
- `https://es.example.com` → **200/401** (reaches Elasticsearch; 401 without credentials, full cluster info with credentials) ✓
- `http://es.example.com` → **308 Permanent Redirect** (plain HTTP does not succeed) ✓
