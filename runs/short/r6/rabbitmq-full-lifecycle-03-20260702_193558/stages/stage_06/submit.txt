**What the upgrade required (key lessons):**

1. **Two-hop upgrade path**: 3.12 → 3.13 → 4.1. RabbitMQ 4.x requires upgrading from 3.13 first — direct 3.12→4.x fails with `incompatible_feature_flags`.

2. **Feature flags must be enabled before each hop**: Run `rabbitmqctl enable_feature_flag all` on the fully-healthy cluster before patching the image.

3. **ConfigMap changes for 4.x compatibility**: `queue_master_locator` was renamed to `queue_leader_locator` in 4.x (causes boot failure if left as-is).

4. **emptyDir + rolling upgrades cause Mnesia loss**: With ephemeral volumes, nodes occasionally form standalone clusters on restart. Used staged partition updates (`partition: 2 → 1 → 0`) and manual `stop_app + reset + join_cluster + start_app` to re-attach any node that formed a standalone cluster.

5. **Definition backup/restore**: Exported definitions at the start and re-imported after any data loss during the rolling update.
