All conditions are met. Here's a summary of what was done and why:

**Root causes found:**

| Issue | What was wrong |
|-------|---------------|
| `app-user` on `/app` | `configure` permission was empty (`""`) — connection could authenticate but queue operations failed |
| `ops-user` on `/app` | Had full `.*`/`.*`/`.*` permissions on the wrong vhost |
| `ops-user` on `/ops` | Had **no permissions at all** — couldn't connect |
| `app-queue` in `/app` | Existed with `x-queue-mode=lazy` but client declared it without that arg → `bad_request` inequivalent arg error |

**Fixes applied:**

1. `rabbitmqctl set_permissions -p /app app-user ".*" ".*" ".*"` — added missing configure permission
2. `rabbitmqctl clear_permissions -p /app ops-user` — removed ops-user's cross-vhost access
3. `rabbitmqctl set_permissions -p /ops ops-user ".*" ".*" ".*"` — granted ops-user its own vhost access
4. `rabbitmqctl delete_queue -p /app app-queue` — removed the stale queue with conflicting arguments so app-client could recreate it cleanly

**End state:**
- `app-user`: full `.*`/`.*`/`.*` on `/app` only
- `ops-user`: full `.*`/`.*`/`.*` on `/ops` only
