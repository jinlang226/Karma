## Summary

**Baseline (before changes):**
| Setting | Value |
|---|---|
| `logLevel` | 1 |
| `slowOpThresholdMs` | 200 ms |
| `journalCompressor` | `snappy` (default) |
| Config file mounted | No |
| Auth enforced | No |

**Applied changes:**
| Setting | New Value | Change |
|---|---|---|
| `logLevel` | **2** | +1 from current |
| `slowOpThresholdMs` | **400 ms** | 2× current |
| `journalCompressor` | **zlib** | Switched to approved standard |

**How it was done:**

1. **Updated ConfigMap** `mongod-config` with the target values.

2. **Patched the StatefulSet** to mount the ConfigMap at `/etc/mongod.conf` and the `mongo-keyfile` secret at `/etc/mongo-keyfile/keyfile` (mode 0400), and changed the container command to `mongod --config /etc/mongod.conf`.

3. **Phase 1 rolling restart** with `transitionToAuth: true` — this allowed the 5 members to safely transition from unauthenticated replication to keyfile auth without ever losing quorum, since transition mode accepts both authenticated and unauthenticated inter-member connections during the rollout.

4. **Phase 2 rolling restart** with `transitionToAuth` removed — enforces full keyfile authentication. All 5 members (4 secondaries + 1 primary) are now running with the config file, full auth, and all three settings applied consistently.
