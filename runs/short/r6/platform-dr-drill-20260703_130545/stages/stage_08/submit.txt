Everything is confirmed working. Here's a summary of what was done:

**Root causes fixed:**
1. **Missing MinIO service** — `minio-init` job had run before a `Service` for the MinIO pod existed, so the bucket was never created and ES couldn't reach MinIO. Created `service/minio` (port 9000) to expose the MinIO deployment.
2. **Missing `es-backups` bucket** — Created it with a temporary `minio/mc` pod after the service was up.
3. **No keystore credentials** — ES cannot authenticate to MinIO without `s3.client.default.access_key` / `secret_key` in the Elasticsearch keystore.

**What was configured:**

| Resource | Action |
|---|---|
| `Secret/es-secure-settings` | Created with `s3.client.default.access_key` and `s3.client.default.secret_key` (both `minioadmin`) |
| `StatefulSet/es` | Added `setup-keystore` init container that reads the secret and runs `elasticsearch-keystore add` to populate `/keystore/elasticsearch.keystore`; main container copies it into the config dir at startup |
| `Service/minio` | Created to expose port 9000 so Elasticsearch pods can resolve `minio.elasticsearch.svc.cluster.local` |

**Result:**
- `minio-repo` repository registered (type `s3`, bucket `es-backups`, path-style access, HTTP protocol)
