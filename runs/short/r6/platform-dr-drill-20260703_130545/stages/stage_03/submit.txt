**What was done:**

Created a `ServiceMonitor` CR in the `cockroachdb` namespace:
```yaml
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: cockroachdb
  namespace: cockroachdb
spec:
  selector:
    matchLabels:
      app.kubernetes.io/name: cockroachdb
      app.kubernetes.io/instance: crdb-cluster
  endpoints:
  - port: http
    path: /_status/vars
    interval: 30s
    scrapeTimeout: 10s
```

The existing `Prometheus` CR (`crdb`) already had `serviceMonitorNamespaceSelector: {matchNames: [cockroachdb]}` and `serviceMonitorSelector: {}`, so it automatically picked up the new `ServiceMonitor`.

1. **Prometheus has 6 active targets** — all `UP` — scraping `/_status/vars:8080` on each CockroachDB pod (3 via the headless service, 3 via the public service, both matching the label selector).
