All done. Here's what was done:

**Root cause:** Two file realm secrets existed (`es-file-realm-aggregate` with `ops-user`, `user-provided-file-realm` with `report-user`) but neither was mounted into the StatefulSet. The Elasticsearch pods had empty `users`/`users_roles` files — so no file realm users could authenticate.

**Fix:**
1. Merged both secrets' `users` and `users_roles` into `es-file-realm-aggregate` (ops-user + report-user combined)
2. Patched the `es-cluster` StatefulSet to mount those three files (`users`, `users_roles`, `roles.yml`) from the merged secret into `/usr/share/elasticsearch/config/` via subPath mounts
3. Waited for the rolling restart to complete (all 3 pods updated)
4. Created and seeded the `app-data` index (the precondition seed step had skipped it since auth was broken)

**Result:**
- `report-user` can authenticate and read indices (viewer role) ✓
- `ops-user` can authenticate and has superuser access ✓  
- Cluster is green with all 3 nodes healthy ✓
