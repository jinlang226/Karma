All goals achieved:

- **Cluster is green** with all **3 nodes** joined and **`es-cluster-1` elected as master** (`*`)
- **`cluster.initial_master_nodes` removed** from the configmap — the config now only has `discovery.seed_hosts` for normal peer discovery
- **Rolling restart completed successfully** — each pod restarted one at a time without `cluster.initial_master_nodes`, and the cluster returned to green with 3 nodes each time

**Root cause:** The `cluster.initial_master_nodes` setting was missing from `elasticsearch.yml`. In ES 7.x this is required to bootstrap master election on a fresh cluster that has no persisted cluster state. Without it, nodes discover each other but block indefinitely on "master not discovered yet." Once the cluster state is persisted on disk, this setting is no longer needed and must be removed — leaving it in place would allow a single node to re-bootstrap a split cluster after a restart.
