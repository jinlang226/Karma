An Ingress resource exists for demo.example.com with TLS enabled.
HTTPS traffic to this domain is failing with certificate validation
errors. The application backend is healthy and responding correctly.

To test the ingress from inside the cluster, first derive the ingress
endpoint from the live cluster:
```bash
INGRESS_NODE_IP=$(kubectl get nodes -o jsonpath='{.items[0].status.addresses[?(@.type=="InternalIP")].address}')
INGRESS_HTTPS_PORT=$(kubectl get svc -n ingress-nginx ingress-nginx-controller -o jsonpath='{.spec.ports[?(@.port==443)].nodePort}')
```

Then use the curl-test pod:
```bash
kubectl -n demo exec curl-test -- /tmp/test_ingress.sh \
  $INGRESS_HTTPS_PORT $INGRESS_NODE_IP demo.example.com
```

Or directly with curl:
```bash
kubectl -n demo exec curl-test -- sh -c \
  'curl -v --cacert /tmp/tls/ca.crt \
    --resolve demo.example.com:'"$INGRESS_HTTPS_PORT"':'"$INGRESS_NODE_IP"' \
    https://demo.example.com:'"$INGRESS_HTTPS_PORT"'/'
```

Note: use the CA override (--cacert /tmp/tls/ca.crt) for validation.
openssl is available via the toolbox pod:
```bash
kubectl -n demo exec openssl-toolbox -- openssl version
```
