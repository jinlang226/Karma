The message count grew from 17 to 37, confirming the producer is actively publishing. Everything is healthy.

---

**Root cause:** `app-user` was missing the **configure** permission on the `/app` vhost. RabbitMQ requires configure permission matching the resource name to declare (create) a queue — write/read alone is not enough. The permission entry was:

```
user       configure   write   read
app-user   (empty)     .*      .*
```

**Fix applied:**
```bash
rabbitmqctl set_permissions -p /app app-user ".*" ".*" ".*"
```
