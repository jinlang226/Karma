All goals are met:

**Root cause:** `app-user` had no **configure** permission on the `/app` vhost — only write and read. RabbitMQ requires the configure permission to declare (create) queues, so every attempt to create `app-queue` was rejected with `Access refused`.

**Fix applied:**
```
rabbitmqctl set_permissions -p /app app-user ".*" ".*" ".*"
```

**Verification:**
- `app-producer` — `1/1 Running`, 0 restarts
- `/app` vhost — `app-queue` of type `classic` exists with 45+ messages
- Cluster — all 3 nodes (`rabbitmq-0/1/2`) running, no alarms, no network partitions
