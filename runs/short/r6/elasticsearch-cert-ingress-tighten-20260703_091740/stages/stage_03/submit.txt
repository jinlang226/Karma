Everything is working correctly:

- **HTTPS** (`https://es.example.com`) → returns 200 with the Elasticsearch cluster info
- **HTTP** (`http://es.example.com`) → returns 308 Permanent Redirect to HTTPS, never serving data

Here's what was created:

1. **TLS secret `es-ingress-tls`** in the `elasticsearch` namespace — a self-signed cert for `es.example.com`, used by ingress-nginx for TLS termination toward clients.

2. **Ingress `elasticsearch-ingress`** in `elasticsearch` namespace with:
   - `ingressClassName: nginx` — uses the existing `nginx` ingress class
   - `nginx.ingress.kubernetes.io/backend-protocol: "HTTPS"` — tells nginx the backend (`es-http:9200`) speaks HTTPS
   - `nginx.ingress.kubernetes.io/proxy-ssl-verify: "off"` — skips backend cert verification (the ES cert is self-signed for internal DNS names, not `es.example.com`)
   - `nginx.ingress.kubernetes.io/ssl-redirect: "true"` + `force-ssl-redirect: "true"` — ensures plain HTTP never succeeds, always redirects to HTTPS
