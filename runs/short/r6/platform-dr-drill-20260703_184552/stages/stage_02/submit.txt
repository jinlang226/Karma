The CockroachDB cluster is fully operational. Here's a summary of what I found and the current state:

**Diagnosis:** When I checked, the cluster had already self-initialized. The initial join/advertise errors you'd see on a fresh StatefulSet are transient — the pods use DNS-based headless service discovery (`crdb-cluster-N.crdb-cluster.cockroachdb.svc.cluster.local`) and `cockroach init` had been run on node 0, which bootstrapped the cluster. Nodes 1 and 2 joined automatically.

**Current state (verified):**

| Node | Address | is_available | is_live |
|------|---------|-------------|---------|
| 1 | crdb-cluster-0 | true | true |
| 2 | crdb-cluster-2 | true | true |
| 3 | crdb-cluster-1 | true | true |

- All 3 nodes are healthy and visible in cluster status
- SQL connections work (`SHOW DATABASES` returns `defaultdb`, `postgres`, `system`)
- The cluster is accepting SQL connections via the `crdb-cluster-public` service on port 26257

The cluster is SQL-ready. You can connect to it via:
```bash
kubectl -n cockroachdb exec -it crdb-cluster-0 -- ./cockroach sql --insecure
```
