The 3-node CockroachDB cluster is fully deployed and healthy. Here's a summary of what was created:

**Resources deployed:**
- `crdb-cluster` — headless Service (ClusterIP: None) for pod DNS, exposing grpc/26257 and http/8080
- `crdb-cluster-public` — regular ClusterIP Service for client access
- `crdb-cluster` StatefulSet — 3 replicas, `cockroachdb/cockroach:v24.1.0`, insecure mode, `podManagementPolicy: Parallel` (required so all 3 pods start simultaneously and can join each other)
- 3× PVCs `datadir-crdb-cluster-{0,1,2}` — 10Gi each, all Bound
- `crdb-cluster-pdb` PodDisruptionBudget — `minAvailable: 2` ensures quorum is preserved during voluntary disruptions

**Key fix:** The StatefulSet uses `podManagementPolicy: Parallel` instead of the default `OrderedReady`. With `OrderedReady`, pod 0 must become Ready before pods 1 and 2 are created — but pod 0 waits to join them, creating a deadlock. `Parallel` starts all 3 simultaneously, then a single `cockroach init` bootstraps the cluster.

All 3 nodes report `is_available: true` and `is_live: true`.
