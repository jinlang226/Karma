Everything is fixed and verified:

**Root cause:** `app-user` had empty `configure` permissions on the `/app` vhost, which blocked it from declaring the `app-queue`. RabbitMQ's management API returned `"Access refused."` on the `PUT /queues/%2Fapp/app-queue` call.

**Fix:** Granted `app-user` full configure/write/read permissions on the `/app` vhost:
```
rabbitmqctl set_permissions -p /app app-user ".*" ".*" ".*"
```

- `app-producer` — 1/1 Running, 0 restarts, publishing continuously
- `/app` vhost — `app-queue` of type `classic` exists with 56+ messages and growing
- RabbitMQ cluster — all 3 nodes healthy (`rabbitmq-0/1/2` all Running)
