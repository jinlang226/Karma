Determine which feature set the cluster is currently operating under
and record it in ConfigMap crdb-version-report under key db_version.

Useful commands:
```bash
kubectl -n cockroachdb exec crdb-cluster-0 -- \
  ./cockroach sql --insecure -e "SHOW CLUSTER SETTING version;"
kubectl -n cockroachdb exec crdb-cluster-0 -- \
  ./cockroach sql --insecure -e "SELECT version();"
```

Verification:
1. ConfigMap recorded:
   kubectl -n cockroachdb get configmap crdb-version-report -o yaml
2. Cluster version:
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach sql --insecure -e "SHOW CLUSTER SETTING version;"
3. Binary version:
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach sql --insecure -e "SELECT version();"
4. Preserve downgrade option:
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach sql --insecure -e "SHOW CLUSTER SETTING cluster.preserve_downgrade_option;"
