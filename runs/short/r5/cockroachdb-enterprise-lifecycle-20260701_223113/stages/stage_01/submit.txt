The 3-node CockroachDB cluster is fully deployed and healthy. Here's a summary of what was created:

**Resources deployed:**
- **Headless Service** `crdb-cluster` — `clusterIP: None`, selects on the required label pair, exposes ports 26257 (grpc) and 8080 (http). `publishNotReadyAddresses: true` so DNS resolves during startup.
- **Public Service** `crdb-cluster-public` — ClusterIP service on the same ports for client access.
- **StatefulSet** `crdb-cluster` — 3 replicas, `podManagementPolicy: Parallel` (required so all 3 nodes start simultaneously before the cluster is initialized), image `cockroachdb/cockroach:v24.1.0`, insecure mode, each pod advertises itself via its FQDN and joins all 3 peer DNS names.
- **PodDisruptionBudget** `crdb-cluster-pdb` — `minAvailable: 2`, ensuring quorum is preserved during voluntary disruptions.
- **3 PVCs** — each `10Gi` RWO, all Bound.

All 3 nodes report `is_available=true` and `is_live=true`.
