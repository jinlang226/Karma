Everything is working. Here's a summary of what was configured:

**HTTPS DB Console (crdb-ui.example.com:443)**
- Created a self-signed TLS secret `crdb-ui-tls` in the `cockroachdb` namespace
- Created an `Ingress` resource (`crdb-ui`) with `ingressClassName: nginx`, TLS terminated at the ingress, and `nginx.ingress.kubernetes.io/backend-protocol: "HTTPS"` so nginx re-encrypts to the backend (CockroachDB's UI is HTTPS-only on port 8080)
- Returns HTTP 200 — confirmed working

**SQL TCP on port 26257**
- Created ConfigMap `tcp-services` in `ingress-nginx` with entry `"26257": "cockroachdb/crdb-cluster-public:26257"`
- Patched the controller deployment to add `--tcp-services-configmap=$(POD_NAMESPACE)/tcp-services`
- Patched the `ingress-nginx-controller` Service to expose port 26257
- nginx.conf now has a dedicated TCP `server` block listening on 26257 proxying to the CockroachDB SQL service — confirmed in config
