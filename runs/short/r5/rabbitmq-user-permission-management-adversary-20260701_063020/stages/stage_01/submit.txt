All goals confirmed:

- **Root cause**: `app-user` had an empty `configure` permission on the `/app` vhost, so it was authorized to `write`/`read` but not to declare (create) queues via the management API — causing the `not_authorised` error and CrashLoop.
- **Fix**: `rabbitmqctl set_permissions -p /app app-user "^app-.*" ".*" ".*"` — scoped `configure` to the app's queue naming convention rather than granting blanket `.*` access.
- **Verified**: `app-producer` pod is `1/1 Running`, `/app` vhost has `app-queue` of type `classic` with messages flowing (10 and counting), and the cluster still shows all 3 nodes running healthy. No CRDs/operators used, no data touched or lost.
