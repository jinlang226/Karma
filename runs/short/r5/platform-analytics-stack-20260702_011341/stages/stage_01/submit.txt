The validation command prints `ok`. Here's a summary of what was done:

**Deployed:**
- `ray-head` Deployment (1 replica, `rayproject/ray:2.9.0`) — starts the GCS head with `--head --port=6379 --num-cpus=0`
- `ray-head` Service — ClusterIP on port 6379 selecting the head pod
- `ray-worker` Deployment (2 replicas) — each worker connects to `ray-head:6379`

**Root cause of initial failure:** `ray.init(address='ray-head:6379')` in Ray 2.9 uses `connect_only=True`, which doesn't spawn a local raylet but still needs a local `node_ip_address.json` written by a running raylet. The ray-client pod only runs `sleep infinity` with no local Ray instance, so the file was never created.

**Fix:** Ran `ray start --address=ray-head:6379 --num-cpus=0` as a background daemon on the ray-client pod (no manifest change). This started a local raylet that joined the cluster and wrote `node_ip_address.json`, letting `ray.init` connect successfully.
