HTTPS is working. The curl returns exit code 0 and prints "hello". Here's what was done:

1. **Root cause**: The adversary (`delete_tls_secret`) had deleted the `expired-tls-secret` that the Ingress referenced for TLS termination, causing certificate validation errors (no secret = no cert served).

2. **Additional issue**: The original cert was intentionally expired (valid only until Jan 2020), which would also cause validation failures even if the secret existed.

3. **Fix**: Generated a new TLS certificate (valid 2026–2028) for `demo.example.com` signed by the same test CA (`/tmp/test-ca.key` + `/tmp/test-ca.crt`), then recreated the `expired-tls-secret` in the `demo` namespace. The Ingress already referenced the correct secret name and the curl-test pod already trusted the CA via `/tmp/tls/ca.crt`.
