The CockroachDB cluster is fully operational:

**Root cause:** An adversary process scaled the StatefulSet from 3 replicas down to 1, causing pods `crdb-cluster-1` and `crdb-cluster-2` to terminate and leaving `crdb-cluster-0` unable to form quorum.

**Fix applied:** Scaled the StatefulSet back to 3 replicas (`kubectl scale statefulset crdb-cluster --replicas=3`). The PVCs were retained (Kubernetes `whenScaled: Retain` policy), so all nodes rejoined as pre-existing members of the same cluster (clusterID `b665bc8b`).

**Final state:**
- All 3 pods: `1/1 Running`
- All 3 nodes: `is_available=true`, `is_live=true`
- SQL accepting connections — `SHOW DATABASES` returns `defaultdb`, `postgres`, `system`
- StatefulSet replicas: 3
- Cluster initialized and healthy (no `cockroach init` needed — cluster was pre-existing)
