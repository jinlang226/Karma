All goals are met:

- **app-producer**: `1/1 Running`, publishing continuously (0 restarts).
- **`/app` vhost**: contains `app-queue`, type `classic`, with messages (11 and growing).
- **Cluster**: all 3 nodes (`rabbitmq-0/1/2`) running and part of the cluster, no data loss — this was a permissions fix only.

**Root cause:** `app-user` had `configure` permission set to an empty string on the `/app` vhost (write/read were fine), so the management-API `PUT /queues/%2Fapp/app-queue` call to declare the classic queue was rejected with `not_authorised`. Fixed with:

```
rabbitmqctl set_permissions -p /app app-user ".*" ".*" ".*"
```

No CRDs/operators were touched, no data was deleted — only the RabbitMQ permission for the existing user was corrected.
