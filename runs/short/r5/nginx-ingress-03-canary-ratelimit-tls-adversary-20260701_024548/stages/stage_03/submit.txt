HTTPS is fully restored. The curl returns exit code 0 and prints "hello".

**Root cause:** The `expired-tls-secret` contained a TLS certificate that was only valid January 1–2, 2020 — a certificate that expired over 6 years ago.

**Fix applied:**
1. Retrieved the CA certificate (`CN=Test CA`) from the curl-test pod at `/tmp/tls/ca.crt`
2. Located the matching CA private key at `/tmp/test-ca.key` on the host (confirmed by matching moduli)
3. Generated a new 2048-bit RSA key + CSR for `demo.example.com` with the correct SAN
4. Signed a new certificate valid July 2026–June 2028 using the CA
5. Updated `expired-tls-secret` in the `demo` namespace with the new cert/key pair

The nginx ingress controller picked up the renewed secret automatically, and TLS handshake now succeeds with `SSL certificate verify ok`.
