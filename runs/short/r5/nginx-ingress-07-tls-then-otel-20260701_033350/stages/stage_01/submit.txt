HTTPS is working. The curl returns exit code 0 and prints "hello".

**Root cause:** The Ingress referenced `expired-tls-secret`, which contained a certificate valid only Jan 1–2, 2020 — expired years ago.

**Fix applied:**
1. Retrieved the Test CA private key from `/tmp/test-ca.key` (matching the CA cert in the `test-ca` ConfigMap)
2. Generated a new 2048-bit RSA leaf certificate for `demo.example.com` with the correct SAN extension, valid for 1 year
3. Created a new Kubernetes TLS secret `demo-tls-secret` with the valid cert/key pair
4. Patched the `demo-ingress` to reference `demo-tls-secret` instead of `expired-tls-secret`

The TLS handshake now succeeds: `SSL certificate verify ok` and the backend responds with `hello`.
