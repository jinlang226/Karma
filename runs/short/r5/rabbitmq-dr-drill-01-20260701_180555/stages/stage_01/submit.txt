Everything is healthy. Here's what was wrong and what I fixed:

**Root cause:** `app-user` had no **configure** permission on the `/app` vhost. In RabbitMQ, declaring (creating) a queue requires configure permission — write permission alone only allows publishing to existing queues. The permissions were:

```
user      configure   write   read
app-user  (empty)     .*      .*
```

**Fix:** Granted `app-user` full permissions on `/app`:
```bash
rabbitmqctl set_permissions -p /app app-user '.*' '.*' '.*'
```

**Verified state:**
- `app-producer` is Running with 0 restarts (no CrashLoop)
- `/app` vhost contains `app-queue` of type `classic` with 70 messages (and growing — the producer is actively publishing)
- All 3 RabbitMQ nodes (`rabbitmq-0/1/2`) are running and in cluster
