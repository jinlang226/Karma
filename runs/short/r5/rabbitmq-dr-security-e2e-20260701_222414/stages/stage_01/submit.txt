**Root cause:** `app-user` was missing the `configure` permission on the `/app` vhost (it had write/read but not configure). Declaring a queue via the RabbitMQ HTTP API requires configure permission on the vhost.

**Fix applied:** `rabbitmqctl set_permissions -p /app app-user '.*' '.*' '.*'` — granted full configure/write/read access.

- `app-producer` is `1/1 Running`, publishing messages successfully
