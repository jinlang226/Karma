**What was done:**

Created a `ServiceMonitor` (Prometheus Operator CRD) in the `cockroachdb` namespace:

```yaml
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: cockroachdb
  namespace: cockroachdb
spec:
  selector:
    matchLabels:
      app.kubernetes.io/name: cockroachdb   # matches both crdb-cluster services
  endpoints:
  - port: http        # port 8080
    path: /_status/vars
    interval: 30s
    scrapeTimeout: 10s
```

The Prometheus CR (`prometheus-crdb`) was already configured with `serviceMonitorNamespaceSelector: matchNames: [cockroachdb]` and `serviceMonitorSelector: {}` (match all), so the new ServiceMonitor was picked up automatically — no changes to the Prometheus CR were needed.
