All goals are met:

- **app-producer** is `1/1 Ready`, no more CrashLoopBackOff.
- **`/app` vhost** has `app-queue` (type `classic`) with messages flowing (5 and growing).
- **Cluster** shows all 3 nodes (`rabbitmq-0/1/2`) running and healthy.

**Root cause:** `app-user` had an empty `configure` permission regex on the `/app` vhost (write/read were `.*`, configure was blank), so the management-API `PUT /queues/%2Fapp/app-queue` call was rejected with `not_authorised`, causing the producer's script to exit and CrashLoop.

**Fix:** granted `app-user` configure rights scoped to the exact queue name:
```
rabbitmqctl set_permissions -p /app app-user "^app-queue$" ".*" ".*"
```
This is least-privilege (only lets the user declare/configure `app-queue`, not arbitrary topology), required no CRD/operator installation, and caused no data loss — the vhost, users, and existing cluster state were untouched.
