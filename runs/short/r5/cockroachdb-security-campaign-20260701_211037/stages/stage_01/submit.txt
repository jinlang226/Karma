The CockroachDB cluster is fully deployed and healthy. Here's a summary of what was deployed:

**Resources created:**
- `Service/crdb-cluster` — headless service (ClusterIP: None) for pod DNS, ports 26257/8080
- `Service/crdb-cluster-public` — regular ClusterIP service for client access, ports 26257/8080
- `StatefulSet/crdb-cluster` — 3 replicas, `podManagementPolicy: Parallel` (required so all nodes start simultaneously to form quorum), image `cockroachdb/cockroach:v24.1.0`, insecure mode
- `PodDisruptionBudget/crdb-cluster-pdb` — `minAvailable: 2` ensures voluntary disruptions never drop below quorum
- 3× `PersistentVolumeClaim` (10Gi each, all Bound)

**Cluster state:**
- All 3 nodes `is_available=true, is_live=true`
- Each advertises via its full DNS name (`crdb-cluster-{0,1,2}.crdb-cluster.cockroachdb.svc.cluster.local`)

**Note:** After deploying a fresh CockroachDB cluster, `cockroach init` must be run once to initialize the cluster — this is a one-time bootstrap step, not a sign of misconfiguration.
