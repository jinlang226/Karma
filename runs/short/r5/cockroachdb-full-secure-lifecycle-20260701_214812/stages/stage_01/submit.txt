The 3-node CockroachDB cluster is fully deployed and healthy:

- **StatefulSet** `crdb-cluster`: 3/3 ready, image `cockroachdb/cockroach:v24.1.0`, insecure mode
- **Headless Service** `crdb-cluster`: ClusterIP None, ports 26257 (grpc) + 8080 (http)
- **Public Service** `crdb-cluster-public`: ClusterIP `10.96.229.189`, same ports
- **PVCs**: 3 × 10Gi `Bound` volumes, one per pod
- **PodDisruptionBudget** `crdb-cluster-pdb`: `minAvailable: 2` — ensures quorum is preserved during voluntary disruptions

All pods carry both required labels (`app.kubernetes.io/name: cockroachdb`, `app.kubernetes.io/instance: crdb-cluster`), and both services select on that label pair. The cluster was bootstrapped with `cockroach init` after all nodes joined.
