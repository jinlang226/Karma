**Root cause:** The `demo-ingress` was configured to use `expired-tls-secret`, which held a leaf certificate with validity window Jan 1–2 2020 (long expired).

**Fix:** Used the CA private key (`/tmp/test-ca.key`) generated during precondition to sign a fresh leaf certificate for `demo.example.com` valid through 2036, then replaced the `expired-tls-secret` data in-place. The ingress configuration itself was correct; only the certificate needed renewal.

**Verification:** TLS handshake succeeds with `SSL certificate verify ok`, certificate matches `demo.example.com` SAN, and the backend returns `hello` with HTTP 200.
