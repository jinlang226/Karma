Both endpoints are working. Here's a summary of what was configured:

**DB Console (HTTPS)**
- Generated a self-signed TLS cert for `crdb-ui.example.com` → stored as secret `crdb-ui-tls` in `cockroachdb`
- Created `Ingress/crdb-ui` (class: `nginx`) with TLS termination, routing `https://crdb-ui.example.com/` → `crdb-cluster-public:8080`

**SQL (TCP port 26257)**
- Created `ConfigMap/tcp-services` in `ingress-nginx` mapping `26257 → cockroachdb/crdb-cluster-public:26257`
- Patched the ingress-nginx controller deployment to load it via `--tcp-services-configmap=ingress-nginx/tcp-services`
- Added port 26257 to the `ingress-nginx-controller` Service

**NetworkPolicy fix**
- The existing `block-crdb-sql` NetworkPolicy only permitted port 8080 ingress. Patched it to also allow port 26257 from the `ingress-nginx` namespace, enabling nginx to forward SQL connections through to the pods.
