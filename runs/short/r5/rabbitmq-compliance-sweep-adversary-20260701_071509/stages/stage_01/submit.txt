- **`/app` vhost contains `app-queue`** of type `classic` with **12 messages** (and growing)
- **3-node cluster healthy** — all 3 rabbitmq pods Running

**Root cause:** `app-user` had empty configure permissions on the `/app` vhost (`"" ".*" ".*"`). Declaring a queue via the Management API requires configure rights matching the queue name. The fix was a single `rabbitmqctl set_permissions` call to grant `app-user` full configure (`".*"`) alongside the existing write/read permissions on `/app`.
