**What was done:**

1. `/opt/job.py` was mounted read-only from a ConfigMap (`ray-job-script`) with the wrong address `ray://ray-head:10001` — a Ray Client protocol address that wasn't exposed by the Service.

2. The `ray-head` Service only exposes port 6379 (GCS), and connecting with `ray.init(address="ray-head:6379")` requires a local Ray raylet to be running on the client pod first (it waits for a local `node_ip_address.json`).

3. Updated the ConfigMap so the script first calls `subprocess.run(["ray", "start", "--address=ray-head:6379"])` to join the client pod to the cluster as a worker node, then calls `ray.init(address="ray-head:6379")` which finds the running local raylet and connects.

4. Deleted and recreated the pod so the new ConfigMap content was picked up (subPath mounts don't live-propagate).
