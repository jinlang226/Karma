A 5-node CockroachDB cluster is running in namespace cockroachdb.

OBSERVED STATE:
- 5 nodes are live.
- A critical table (bench.decom_data) has replicas concentrated on the highest-ordinal nodes.

YOUR TASK:
Safely reduce the cluster to 3 nodes without losing data or availability.

CONSTRAINTS:
- Do not delete or recreate the cluster.
- Keep bench.decom_data readable throughout.
- Use CockroachDB's decommission process for nodes being removed.

GOAL:
- StatefulSet scaled to 3 replicas.
- Removed nodes fully decommissioned.
- Cluster remains SQL-ready.

Useful commands:
```bash
kubectl -n cockroachdb exec crdb-cluster-0 -- ./cockroach node status --insecure
kubectl -n cockroachdb exec crdb-cluster-0 -- ./cockroach node decommission <id> --insecure
kubectl -n cockroachdb exec crdb-cluster-0 -- ./cockroach sql --insecure -e "SELECT count(*) FROM bench.decom_data;"
```

Verification:
1. Verify 3 nodes in cluster:
   kubectl -n cockroachdb exec crdb-cluster-0 -- ./cockroach node status --insecure
   (Should show 3 live nodes)

2. Check StatefulSet replicas:
   kubectl -n cockroachdb get sts crdb-cluster
   (Should show 3/3 ready)

3. Verify data still available:
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach sql --insecure -e "SELECT count(*) FROM bench.decom_data;"
   (Should return the seeded rows)
