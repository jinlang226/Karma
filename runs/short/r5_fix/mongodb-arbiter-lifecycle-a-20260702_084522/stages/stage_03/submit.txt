Done. Here's what was done and why it works:

**Problem:** `rs0` had 2 voting members (`majorityVoteCount: 2`), meaning both had to agree for any election to succeed — losing either member meant no primary could be elected.

**Fix:** Added `mongo-arb-0` as an **arbiter** member via `rs.addArb()`.

**Result:**
- `votingMembersCount: 3` — now has an odd number of voters
- `majorityVoteCount: 2` — only 2 of 3 needed for a quorum
- The arbiter (state `ARBITER`) votes in elections but stores no data, so there's **zero storage cost increase**
- Either data-bearing member can now be lost and elections still succeed (the surviving data node + arbiter form a majority)
- `app.test` data (3 documents) is **intact** — untouched

The `setDefaultRWConcern` call was needed first because MongoDB 5+ requires an explicit cluster-wide write concern before a reconfig that changes the implicit majority behavior.
