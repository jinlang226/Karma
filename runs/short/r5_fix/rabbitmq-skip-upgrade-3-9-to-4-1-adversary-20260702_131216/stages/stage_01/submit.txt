All goals are met. Here's a summary of what was wrong and what was fixed:

**Issues found:**

1. **StatefulSet scaled down** — An external process had reduced the StatefulSet replicas from 3 to 1, causing rabbitmq-1 and rabbitmq-2 to be terminated. Fixed by scaling back to 3.

2. **`rabbit` app stopped on rabbitmq-0** — The node's rabbit application was not running (likely stopped during the scale-down disruption). Fixed by running `rabbitmqctl start_app`.

3. **Missing configure permission for `app-user` on `/app` vhost** — The user had write (`.*`) and read (`.*`) but no configure permission, so it couldn't declare the `app-queue` classic queue. Fixed by:
   ```
   rabbitmqctl set_permissions -p /app app-user '.*' '.*' '.*'
   ```

**Current state:**
- `app-producer` is `1/1 Running` with 0 restarts
- `/app` vhost has `app-queue` (classic) with 10 messages
- All 3 RabbitMQ nodes (`rabbitmq-0`, `rabbitmq-1`, `rabbitmq-2`) are running and in the cluster
