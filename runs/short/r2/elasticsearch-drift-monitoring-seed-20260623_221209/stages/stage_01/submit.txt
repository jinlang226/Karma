Everything is working. Here's a summary of what was deployed and the two issues that were fixed along the way:

---

**Deployed resources:**

| Resource | Name | Purpose |
|---|---|---|
| ConfigMap | `elasticsearch-config` | `elasticsearch.yml` with TLS + security settings |
| Service (headless) | `elasticsearch-headless` | Pod-to-pod transport discovery on 9300/9200 |
| Service (ClusterIP) | `es-http` | HTTPS access at port 9200 |
| PodDisruptionBudget | `elasticsearch-pdb` | `minAvailable: 2` |
| StatefulSet | `elasticsearch` | 3 replicas, `podManagementPolicy: Parallel` |

**Two issues fixed mid-deployment:**

1. **`publishNotReadyAddresses: true`** on the headless Service — without this, Kubernetes omits not-ready pods from DNS, so nodes couldn't discover each other via `elasticsearch-headless` until at least one was already healthy (a DNS bootstrapping deadlock).

2. **`podManagementPolicy: Parallel`** on the StatefulSet — the default `OrderedReady` only creates the next pod once the previous is Ready, but ES needs a 2/3 quorum to elect a master before any node becomes Ready. All 3 must start simultaneously.

**Final state:** Cluster is `green`, 3/3 nodes joined, master on `elasticsearch-2`, reads and writes succeed over HTTPS through `es-http:9200`.
