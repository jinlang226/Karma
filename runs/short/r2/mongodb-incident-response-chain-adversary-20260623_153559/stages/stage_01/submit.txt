**Root cause:** The `health-overrides` ConfigMap contained a stale entry for `mongodb-replica-1` with password `legacyHealthPass123`, but the actual `health-user` in MongoDB uses `healthpass123` (from the `health-user-password` Secret). The readiness probe was authenticating with the wrong password, causing repeated auth failures.

**Fix:** Removed the stale `mongodb-replica-1` key from the `health-overrides` ConfigMap so the probe falls back to the correct password from the Secret.
