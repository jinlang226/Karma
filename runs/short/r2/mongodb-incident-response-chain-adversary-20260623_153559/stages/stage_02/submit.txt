All pods are stable. The fix was straightforward:

**Root cause:** The `health-overrides` ConfigMap contained a stale entry for `mongodb-replica-1` with password `legacyHealthPass123`. The health check script uses per-pod override files when present, so `mongodb-replica-1`'s probe was authenticating with the wrong password while `healthpass123` (from the `health-user-password` secret) was the actual credential.

**Fix:** Removed the stale override entry from the `health-overrides` ConfigMap so all pods fall back to the correct password from the secret.

**Result:**
- All 3 pods: `1/1 Running` with no probe failures
- Replica set: 1 PRIMARY (`mongodb-replica-1`) + 2 SECONDARY (`mongodb-replica-0`, `mongodb-replica-2`)
