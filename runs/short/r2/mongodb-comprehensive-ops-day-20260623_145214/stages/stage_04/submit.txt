TLS is fully enabled and enforced. Here's what was done:

**Certificates generated** (via openssl-toolbox pod):
- `ca.crt` — self-signed CA cert (4096-bit RSA, 10-year validity)
- `server.pem` — server key+cert with SANs covering all replica set member hostnames (`mongo-rs-{0,1,2}.mongo.mongodb.svc.cluster.local`, `localhost`, `127.0.0.1`)
- `client.pem` — client key+cert signed by the same CA

**Secret** `mongo-tls` created in the `mongodb` namespace with all three files.

**StatefulSet patched** to:
- Mount the `mongo-tls` secret at `/etc/tls` on every pod
- Pass `--tlsMode requireTLS --tlsCertificateKeyFile /etc/tls/server.pem --tlsCAFile /etc/tls/ca.crt` to `mongod`

**Verification results:**
- Plain connections → `MongoServerSelectionError` (refused, as required)
- TLS connections with `/etc/tls/client.pem` + `/etc/tls/ca.crt` → `ok: 1`
- Replica set remains healthy: 3 members (1 PRIMARY, 2 SECONDARY)
