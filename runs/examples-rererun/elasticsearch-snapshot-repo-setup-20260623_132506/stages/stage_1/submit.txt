- The `elasticsearch.yml` ConfigMap had the S3 endpoint, protocol, and path-style access already configured (no credentials there, as required)

**Actions taken:**
1. Registered the `minio-repo` snapshot repository using the S3 type, pointing at the `es-backups` bucket with the `default` S3 client (which picks up credentials from the keystore)
2. Created `snapshot-1` — completed with `state: SUCCESS` in ~1 second
