The cluster is fully operational. Here's a summary of what happened and what was done:

**Root cause:** The 3-node CockroachDB StatefulSet was correctly configured with proper `--join` flags and `--advertise-host` settings, but the cluster had never been bootstrapped. All nodes start in a "waiting for init" state and won't form a cluster until one node receives the one-time `cockroach init` command.

**Fix:** Ran `cockroach init --insecure --host=localhost:26257` on `crdb-cluster-0`. This bootstraps node 1 as the first member, after which nodes 2 and 3 automatically joined using the pre-configured `--join` list.

**Result:**
- All 3 pods are `1/1 Ready`
- All 3 nodes show `is_available=true`, `is_live=true`
- Cluster accepts SQL connections and all 3 nodes are visible in gossip
