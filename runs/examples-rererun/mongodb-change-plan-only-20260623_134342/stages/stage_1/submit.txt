**The plan (6 phases, no changes applied):**

1. **Credential hardening** — rotate the weak static password, split the super-user into three separate accounts (cluster-admin, user-admin, DBA), add a read-only monitoring user.
2. **Write concern** — change the replica-set default to `w: majority, wtimeout: 5000` to survive primary failovers.
3. **mongod.conf tuning** — restore `slowOpThresholdMs` to `100 ms`, drop verbosity to `0`, restrict `bindIp` from all-interfaces to loopback + pod IP. Rolling restart.
4. **Primary preference** — raise `mongodb-replica-0` priority to 2 for deterministic primary placement.
5. **TLS** — add client-facing TLS with a `allowTLS` → `requireTLS` transition to avoid a hard client cutover.
6. **x.509 internal auth** — replace the shared keyFile with per-member certificates, stepped through `sendKeyFile` → `sendX509` → `x509` to avoid disrupting the replica set mid-migration.
