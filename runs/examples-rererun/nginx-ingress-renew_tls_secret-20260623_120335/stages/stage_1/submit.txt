The TLS issue is resolved. Here's what was wrong and what was fixed:

**Root cause:** The Ingress referenced a secret named `expired-tls-secret` containing a TLS certificate that expired on 2020-01-02 (notAfter: `Jan 2 2020`). Any client performing certificate validation would reject the handshake.

**Fix:**
1. Located the test CA at `/tmp/test-ca.crt` / `/tmp/test-ca.key` — this is the same CA whose `ca.crt` is mounted in the `curl-test` pod at `/tmp/tls/ca.crt`.
2. Generated a new RSA 2048 private key and a CSR for `demo.example.com`.
3. Signed the CSR with the test CA, producing a certificate valid from 2026-06-23 to 2027-06-23 with the correct SAN (`DNS:demo.example.com`).
4. Updated `expired-tls-secret` in-place with the new cert/key — no Ingress changes needed since it already references the correct secret name.

The Ingress controller picked up the updated secret and `curl -s --cacert /tmp/tls/ca.crt` now returns `hello` with exit code 0.
