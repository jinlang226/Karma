All deployments are healthy (1/1 ready). The fix is complete:

**Root cause:** The `expired-tls-secret` contained a TLS certificate for `demo.example.com` that expired in January 2020 (it was intentionally backdated to 2020-01-01–2020-01-02 during setup).

**Fix:** Generated a new RSA-2048 private key and CSR using the existing `leaf.cnf` config, then signed it with the Test CA (`/tmp/test-ca.crt` / `/tmp/test-ca.key`) with a 2-year validity (expires 2028). Updated the `expired-tls-secret` in the `demo` namespace with the new cert and key.

**Result:** `curl --cacert /tmp/tls/ca.crt` now succeeds — TLS handshake completes against the Test CA, returns HTTP 200 with body `hello`.
