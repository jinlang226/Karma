Two backend services exist in the demo namespace:
- stable: returns "stable"
- canary: returns "canary"

A stable Ingress routes canary.example.com to the stable backend.
A canary Ingress exists, but canary traffic is not reaching the canary
backend.

Your task:
1) Requests with header "X-Feature: v2" must route to the canary backend.
2) Requests without that header must route to the stable backend.

Do not modify the application Deployments or Services.

Use these requests to test:
```bash
kubectl -n demo exec curl-test -- curl -sS -H "Host: canary.example.com" \
  http://ingress-nginx-controller.ingress-nginx.svc/
```
```bash
kubectl -n demo exec curl-test -- curl -sS -H "Host: canary.example.com" \
  -H "X-Feature: v2" \
  http://ingress-nginx-controller.ingress-nginx.svc/
```

Success criteria:
- Without header: body is "stable".
- With header: body is "canary".

Check controller logs if needed:
```bash
kubectl -n ingress-nginx logs deploy/ingress-nginx-controller --tail=200
```
