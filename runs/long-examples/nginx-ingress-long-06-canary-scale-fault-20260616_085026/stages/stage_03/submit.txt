Both routes work correctly:

- Without header → `stable`
- With `X-Canary: beta` → `canary`

**What was wrong and what I changed:**

1. **`canary-canary` ingress** had `nginx.ingress.kubernetes.io/canary-by-header-value: always` (a catch-all) instead of `beta`. Changed it to `beta` so only requests with exactly `X-Canary: beta` are routed to the canary backend.

2. **`canary-alt` decoy ingress** (labelled `benchmark.decoy: "true"`) was interfering — it claimed `stable-echo` as a `noServer: true` canary backend with header value `beta`. After the first fix introduced a second `beta`-valued canary, the controller could no longer use `stable-echo` as the primary server for `canary.example.com`, breaking all traffic. Deleting the decoy freed `stable-echo` to be the primary backend again, allowing `canary-echo` to be correctly registered as the header-matched alternative.
