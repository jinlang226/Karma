HTTPS is restored. The curl returns exit code 0 and prints "hello".

**Root cause:** The `expired-tls-secret` referenced by the `demo-ingress` TLS configuration contained a certificate valid only from Jan 1–2, 2020 (expired 6+ years ago), causing TLS handshake failures.

**Fix:** Generated a new 2048-bit RSA certificate for `demo.example.com` with a SAN matching the hostname, signed by the cluster's test CA (`/tmp/test-ca.crt`/`test-ca.key`), valid until June 2027, and replaced the secret in-place. The nginx ingress controller picked up the new secret and now serves a valid certificate that the curl-test pod verifies against `/tmp/tls/ca.crt`.
