Everything is working correctly:

- **HTTPS (`https://es.example.com`)** returns HTTP 200 with the Elasticsearch response — the API is fully reachable.
- **Plain HTTP (`http://es.example.com`)** returns HTTP 308 (Permanent Redirect to HTTPS) — the request is rejected/not served as plain HTTP.

Here's what was created:

1. **TLS secret `es-example-com-tls`** (namespace `elasticsearch`) — a self-signed certificate for `es.example.com` generated via the existing `openssl-toolbox` pod.

2. **Ingress `es-ingress`** (namespace `elasticsearch`) with:
   - `ingressClassName: nginx` — uses the nginx ingress class
   - `nginx.ingress.kubernetes.io/backend-protocol: HTTPS` — tells nginx to proxy to the ES backend over HTTPS (since ES has `xpack.security.http.ssl.enabled: true`)
   - `nginx.ingress.kubernetes.io/force-ssl-redirect: "true"` — plain HTTP is permanently redirected to HTTPS, never served
   - `nginx.ingress.kubernetes.io/proxy-ssl-verify: "off"` — skips backend cert validation (the ES cert's SANs don't cover the external hostname)
