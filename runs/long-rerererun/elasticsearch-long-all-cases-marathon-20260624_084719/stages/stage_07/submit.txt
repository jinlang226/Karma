Everything is working correctly. Here's a summary of what was done:

**What was created:**

1. **TLS secret** `es-example-com-tls` in the `elasticsearch` namespace — a self-signed certificate for `es.example.com`.

2. **Ingress** `es-ingress` in the `elasticsearch` namespace with:
   - `ingressClassName: nginx` — uses the nginx ingress class
   - `nginx.ingress.kubernetes.io/backend-protocol: "HTTPS"` — tells nginx to proxy to Elasticsearch over HTTPS (since ES runs TLS internally on port 9200)
   - `nginx.ingress.kubernetes.io/force-ssl-redirect: "true"` + `ssl-redirect: "true"` — ensures plain HTTP returns a **308 Permanent Redirect**, so HTTP access never reaches the backend
   - TLS termination at nginx using the `es-example-com-tls` cert for `es.example.com`

**Verification results:**
- `https://es.example.com` → ✅ Returns Elasticsearch cluster info (green, 3 nodes)
- `http://es.example.com` → ✅ Returns `308 Permanent Redirect` — no data served over plain HTTP
