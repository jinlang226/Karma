Everything is working. Here's a summary of what was done:

**Root cause:** The `demo-ingress` resource was referencing `expired-tls-secret`, which contained a TLS certificate valid only from Jan 1–2, 2020 (expired over 6 years ago). This caused all HTTPS connections to `demo.example.com` to fail certificate validation.

**Fix applied:**
1. **Generated a new TLS certificate** for `demo.example.com` using the current Test CA's key (`/tmp/test-ca.key`) and certificate — valid 2026–2028, with proper SAN for `demo.example.com`.
2. **Updated `expired-tls-secret`** with the new cert/key pair (signed by the same CA that the curl-test pod trusts via `/tmp/tls/ca.crt`).
3. **Scaled `demo-app` back to 1 replica** (it had been scaled to 0).

**Result:** `curl --cacert /tmp/tls/ca.crt --resolve demo.example.com:31548:172.18.0.3 https://demo.example.com:31548/` now returns HTTP 200 with body `hello`.
