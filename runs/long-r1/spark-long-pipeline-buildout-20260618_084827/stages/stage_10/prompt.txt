A Spark cluster is processing workloads with varying load patterns.
YOU must actively monitor and scale the cluster to handle load spikes.

IMPORTANT: The system will NOT auto-scale by itself. YOU must manually scale
the workers using kubectl commands based on the traffic phases.

Traffic Pattern (simulated by traffic-generator):
- Phase 1: Baseline load for 60 seconds (5 workers sufficient)
- Phase 2: 2x load spike for 90 seconds (YOU must scale to 10 workers)
- Phase 3: 5x load spike for 120 seconds (YOU must scale to 20 workers)
- Phase 4: Cooldown for 60 seconds (YOU must scale back to 5 workers)

YOUR REQUIRED ACTIONS:

1. MONITOR traffic-generator logs continuously:
   ```bash
   kubectl logs -n spark-streaming -l app=traffic-generator -f
   ```
   Watch for PHASE transitions and LAG/LATENCY values.

2. SCALE WORKERS based on load phases:

   When you see "PHASE 2" with high LAG (>10K):
   ```bash
   kubectl scale deployment spark-worker -n spark-streaming --replicas=10
   ```

   When you see "PHASE 3" with critical LAG (>50K):
   ```bash
   kubectl scale deployment spark-worker -n spark-streaming --replicas=20
   ```

   When you see "PHASE 4" (cooldown):
   ```bash
   kubectl scale deployment spark-worker -n spark-streaming --replicas=5
   ```

3. VERIFY scaling events are logged:
   ```bash
   kubectl logs -n spark-streaming -l app=metrics-server --tail=50
   ```
   Look for "SCALING EVENT" messages.

Success Criteria (checked by oracle):
- Spark cluster is running (master + workers)
- At least 2 scaling events occurred (YOU must perform these)
- Traffic generator completed or running

Monitoring Commands:
```bash
# Check all pods
kubectl get pods -n spark-streaming

# Check deployments
kubectl get deployments -n spark-streaming

# Check current worker count
kubectl get deployment spark-worker -n spark-streaming -o jsonpath='{.spec.replicas}'

# Watch metrics-server for scaling events
kubectl logs -n spark-streaming -l app=metrics-server --tail=50

# Watch traffic generator for phase changes
kubectl logs -n spark-streaming -l app=traffic-generator --tail=50
```
