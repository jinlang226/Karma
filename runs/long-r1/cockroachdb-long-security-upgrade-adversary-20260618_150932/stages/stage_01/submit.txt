The CockroachDB cluster is deployed and fully operational.

## Summary

A 3-node CockroachDB cluster is running in the `cockroachdb` namespace, using only native Kubernetes resources (no operator, no CRDs) and the pre-existing RBAC (`crdb-cluster-sa`).

**Resources created:**
- **Headless Service** `crdb-cluster` (`clusterIP: None`) — exposes `grpc` (26257) and `http` (8080); provides stable per-pod DNS.
- **Public Service** `crdb-cluster-public` (ClusterIP) — same ports, for client access.
- **StatefulSet** `crdb-cluster` — 3 replicas, image `cockroachdb/cockroach:v24.1.0`, insecure mode.
- **PodDisruptionBudget** `crdb-cluster` (`maxUnavailable: 1`) — guarantees quorum (≥2 of 3 nodes) survives voluntary disruptions.

**Key configuration details:**
- Each pod advertises itself by its stable DNS name (`$(hostname).crdb-cluster.cockroachdb.svc.cluster.local`).
- `--join` lists all 3 node DNS names so any pod can bootstrap into the cluster.
- VolumeClaimTemplate `datadir` → 3× 10Gi PVCs, all `Bound`.
- Ran `cockroach init` once to bootstrap the cluster (required with `--join`).

**Verified end state:**
- StatefulSet `3/3` ready; all 3 pods `1/1 Running`.
- All 3 PVCs `Bound` at 10Gi.
- `cockroach node status` confirms 3 nodes, all live and available.
- PDB shows `ALLOWED DISRUPTIONS: 1`, preserving quorum.

The manifest is saved at `crdb.yaml` in the stage directory.
