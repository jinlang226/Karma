**What happened with the raylet error:** `ray.init(address='...')` in Python tries to start a local raylet process on the calling pod to join the cluster as a node, but it waits for a `node_ip_address.json` file that the raylet writes only after its own GCS registration completes. On the first call this created a stale lock file that blocked retries. Running `ray start --address=ray-head:6379` first brought the client's local raylet up properly, after which `ray.init()` found the already-running session and connected instantly.
