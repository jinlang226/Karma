ConfigMap created and live cluster is unchanged. Here's a summary of what was done and what the script covers:

---

**ConfigMap `rollback-rehearsal`** is in namespace `cockroachdb` with key `rollback.sh`.

**What the script will revert (when executed in the change window):**

| Category | Setting | Current | Default |
|---|---|---|---|
| Cluster setting | `diagnostics.reporting.enabled` | `true` | `false` |
| Cluster setting | `kv.range_merge.queue.enabled` | `false` | `true` |
| Cluster setting | `kv.snapshot_rebalance.max_rate` | `208 MiB` | `32 MiB` |
| Zone config | `RANGE default` `gc.ttlseconds` | `14400` | `90000` |
| Zone config | `TABLE defaultdb.tenant_b.table_c` | custom zone | discard (inherit default) |
| Zone config | `TABLE defaultdb.tenant_b.table_d` | custom zone | discard (inherit default) |

**Deliberately excluded from reset:**
- `cluster.secret` — cluster identity token set at init, resetting it breaks cluster auth
- `version = 24.1` — cluster version, cannot be rolled back via `RESET`
- System range zones (`RANGE meta/system/liveness`, `DATABASE system`, system tables) — these are CockroachDB's own built-in defaults, not user overrides

**StatefulSet replicas** are 3, which is the CockroachDB default — no change needed.
