### One important note
`ray.init(address='ray-head:6379')` connects in driver/connect-only mode — it requires a Ray **node** to already be running on the host executing it. A bare `ray-client` pod (just `sleep infinity`) has no raylet, so the command fails with `ConnectionError` / "node_ip_address.json not found" on its own. For the validation to succeed, `ray-client` must join the cluster as a node, so I ran `ray start --address=ray-head:6379 --num-cpus=0` inside it. This adds a runtime process but does **not** modify the pod's manifest/spec — the head now reports 4 nodes (head + 2 workers + client), still satisfying the oracle's ≥3 requirement. If you'd prefer the client *not* be a cluster node, the alternative would be exposing the Ray Client server (port 10001) and using a `ray://` address, but that would require changing the validation command.
