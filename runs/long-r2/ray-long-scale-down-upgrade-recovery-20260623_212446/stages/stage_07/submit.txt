The job script now runs successfully. Here's what was wrong and what was fixed:

**Root cause:** The bug had two parts:
1. The `ray-head` service was missing the `client` port (10001) — the Ray Client server port was replaced by the dashboard port (8265).
2. The `ray-job-script` ConfigMap had been overwritten with a broken subprocess-based script using `ray-head:6379` (the GCS port, which workers use to join the cluster — not what Python clients use).

**Fix applied:**
1. Restored the `ray-head` service to expose port 10001 (`client`) per the case's `ray-head-service.yaml`.
2. Restored the `ray-job-script` ConfigMap to the correct script: `ray.init(address="ray://ray-head:10001")` — the Ray Client protocol.
3. Recreated the `ray-client` pod to remount the updated ConfigMap.

The command `kubectl -n ray exec ray-client -- python /opt/job.py` now exits 0 and prints `pong`.
