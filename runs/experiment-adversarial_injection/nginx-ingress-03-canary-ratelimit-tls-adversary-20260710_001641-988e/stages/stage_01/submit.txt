Fixed. The `canary-canary` Ingress had `nginx.ingress.kubernetes.io/canary-by-header-value` set to `"wrong"` instead of `"always"`, so nginx never matched the header and fell through to the stable Ingress. I patched the annotation to `always` and verified: requests without the header return `stable`, and requests with `X-Canary: always` return `canary`.
