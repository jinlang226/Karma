Both requirements verified.

**Root cause:** Neither `rate-api` nor `rate-health` ingress had any rate-limiting annotations, so `/api` was unthrottled. (A decoy ingress `rate-api-alt-hard` for a different host was left untouched.)

**Fix:** Annotated only the `rate-api` ingress:
```
nginx.ingress.kubernetes.io/limit-rps: "2"
nginx.ingress.kubernetes.io/limit-burst-multiplier: "1"
```
This generates `limit_req zone=... rate=2r/s burst=2 nodelay;` applied solely to the `/api` location — confirmed via the live nginx.conf, and confirmed `/health` has no `limit_req` directive at all.

**Verified:** Burst traffic to `/api` returns `200 200 200 503 503 ...` (rate-limited after burst is exhausted), while burst traffic to `/health` returns all `200`s. No application Deployments were modified.
