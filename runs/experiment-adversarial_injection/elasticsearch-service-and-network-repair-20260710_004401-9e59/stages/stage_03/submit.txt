Everything is healthy. Summary:

**Root cause found:** the `ingress-nginx-controller` pod was stuck `Pending` — its pod spec requires the node label `ingress-ready=true`, which wasn't set on the single node.

**Changes made:**
1. Labeled the node: `kubectl label node karma-7-control-plane ingress-ready=true` → controller scheduled and became `Running`/`Ready`.
2. Generated a self-signed TLS cert for `es.example.com` and stored it as `Secret/es-tls` in the `elasticsearch` namespace.
3. Created `Ingress/es-http` in `elasticsearch` using `ingressClassName: nginx`, routing `es.example.com` → `es-http:9200`, with TLS configured and `nginx.ingress.kubernetes.io/ssl-redirect: "true"` / `force-ssl-redirect: "true"` so plain HTTP is redirected (308) instead of served.

**Verified:**
- `https://es.example.com/` → `200` with the Elasticsearch API JSON response.
- `http://es.example.com/` → `308` redirect to HTTPS (no API data over plain HTTP).
- Elasticsearch cluster remains `green`, 3/3 nodes, all pods healthy.
