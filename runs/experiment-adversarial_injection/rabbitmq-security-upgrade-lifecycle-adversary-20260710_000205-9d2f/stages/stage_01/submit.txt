- **app-producer**: `1/1 Running`, no more crash loop.
- **/app vhost**: `app-queue` exists, type `classic`, and already has messages (3 and growing, published every 2s).
- **Cluster**: all 3 RabbitMQ nodes (`rabbitmq-0/1/2`) still `Running`, cluster status shows all 3 as running disc nodes — no data loss, no CRDs/operators touched.

**Root cause**: The `rabbitmq-setup` job granted `app-user` write/read permissions on `/app` but left `configure` permission empty (`""`), which blocks queue declaration. Since queue declaration requires `configure` permission, `app-producer`'s `PUT /api/queues/%2Fapp/app-queue` call was rejected with `not_authorised`, causing the crash loop.

**Fix**: Ran `rabbitmqctl set_permissions -p /app app-user ".*" ".*" ".*"` to grant `app-user` configure rights on `/app`, then recycled the producer pod so it could declare `app-queue` (classic type) and resume publishing.
