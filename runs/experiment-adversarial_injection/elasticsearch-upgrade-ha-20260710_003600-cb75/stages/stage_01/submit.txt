Deployed a 3-node Elasticsearch 8.11.0 cluster in the `elasticsearch` namespace using only native resources:

- **es-config** ConfigMap — `elasticsearch.yml` with discovery via the headless service, HTTP TLS from `es-http-tls`, and transport TLS from a newly generated `es-transport-certs` secret (self-signed CA, since node-to-node TLS needs a cert/key that wasn't provided).
- **es-headless** (ClusterIP: None, `publishNotReadyAddresses: true`) for peer discovery, and **es-http** (ClusterIP) exposing port 9200.
- **es-pdb** PodDisruptionBudget (`minAvailable: 2`).
- **es** StatefulSet, 3 replicas, `es-0/1/2`, each with a 5Gi PVC (local-path storage class), init containers to fix data-dir ownership and bump `vm.max_map_count`, and `ELASTIC_PASSWORD` sourced from the `elastic-password` secret.

Verified via the pre-existing `curl-test` pod:
- `_cluster/health` → `status: green`, 3 nodes, 3 data nodes, reached over HTTPS at `https://es-http.elasticsearch.svc.cluster.local:9200` with the CA from `es-http-tls`.
- Write (`POST test-index/_doc/1`), read (`GET test-index/_doc/1`), and search all succeeded, with the index going green (primary + replica both active across nodes), confirming reads/writes work cluster-wide. Test index was cleaned up afterward.
