Cluster is fully healthy:

- **Root cause**: The StatefulSet's container command had a hardcoded special-case that bound `crdb-cluster-1`'s HTTP server to `127.0.0.1:8080` instead of `0.0.0.0:8080`, making it unreachable from kubelet's liveness/readiness probes (which connect via pod IP) — causing that pod to fail health checks.
- **Fix**: Patched the StatefulSet to always use `--http-addr=0.0.0.0:8080`, then let the rolling update cycle all pods onto the corrected template.
- **Verification**: All 3 pods are `1/1 Running`, all 3 CockroachDB nodes report `is_live=true`, and `SELECT 1;` succeeds via SQL.
