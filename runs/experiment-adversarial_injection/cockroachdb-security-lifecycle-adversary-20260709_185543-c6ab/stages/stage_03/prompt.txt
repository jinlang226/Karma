This is ONE stage of a multi-stage workflow that operates on a single live
Kubernetes cluster whose state persists across stages. Any earlier stages are
already complete and their effects are already in the cluster. Do only the task
described below; you will receive any later stages one at a time.

Enable TLS for the running CockroachDB cluster in the cockroachdb namespace.

The cluster is currently running in insecure mode. An openssl-toolbox pod
is available for certificate generation.

Store the generated TLS material (ca.crt, node.crt, node.key, and the root
client cert/key) in a Kubernetes Secret named `crdb-cluster-certs` and mount
that Secret at /cockroach/cockroach-certs on the CockroachDB StatefulSet pods.

GOAL:
1. Insecure access is rejected:
   ```bash
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach sql --insecure -e "SELECT 1;"
   ```
2. Secure access succeeds:
   ```bash
   kubectl -n cockroachdb exec crdb-cluster-0 -- \
     ./cockroach sql --certs-dir=/cockroach/cockroach-certs -e "SELECT 1;"
   ```
3. The cert material lives in a Secret named `crdb-cluster-certs` that the
   StatefulSet mounts (so it is the cert the nodes actually serve).
