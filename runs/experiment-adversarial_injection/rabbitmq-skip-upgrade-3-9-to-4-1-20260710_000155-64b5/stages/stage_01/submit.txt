Everything is confirmed working:

- **app-producer**: `1/1 Running`, no new restarts since the fix (last crash was before the permission change).
- **`/app` vhost**: contains `app-queue` of type `classic`, with message count actively growing (24 → 33), confirming successful publishing.
- **Cluster**: all 3 nodes (`rabbitmq-0/1/2`) are disk nodes and running in the cluster.

**Root cause:** `app-user` had an empty `configure` permission regex on the `/app` vhost (write/read were `.*` but configure was blank), so RabbitMQ refused the queue declaration with `not_authorised`. Fixed via `rabbitmqctl set_permissions -p /app app-user ".*" ".*" ".*"` — no CRDs, no operators, no data loss (existing vhost/queue data untouched).
