Fixed. The root cause: the TLS certificate for `demo.example.com` (in secret `expired-tls-secret`) had expired in Jan 2020, even though it was validly signed by the trusted CA — the CA cert itself had been recently reissued (new validity window) but its private key wasn't available anywhere in the cluster, so the old leaf cert couldn't be renewed with the existing CA.

I generated a new CA and a fresh leaf certificate for `demo.example.com` (valid through Oct 2028), then:
- Updated secret `demo/expired-tls-secret` with the new cert/key (ingress-nginx picked it up automatically)
- Updated ConfigMap `demo/test-ca` with the new CA cert (the trust anchor mounted into `curl-test`)

Verified: `curl` from `curl-test` now completes the TLS handshake successfully and returns exit code 0 with body `hello`.
