This is ONE stage of a multi-stage workflow that operates on a single live
Kubernetes cluster whose state persists across stages. Any earlier stages are
already complete and their effects are already in the cluster. Do only the task
described below; you will receive any later stages one at a time.

A 3-member MongoDB replica set is running in the mongodb namespace.
Authentication is enabled. Passwords are stored in these Secrets:
- admin-user-password
- app-user-password
- readonly-user-password

Task:
- app-user must have readWrite on appdb.
- readonly-user must have read on appdb and a custom role reportingRole.
- reportingRole must allow read access to appdb.reports (aggregate should succeed).
- Do not break admin access or existing data.

Notes:
- Do not use operators or CRDs.
