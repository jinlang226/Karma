Everything is ready:

- `ray-head` deployment: 1/1 ready
- `ray-worker` deployment: 2/2 ready (2 replicas as required)
- `ray-head` Service: exposes port 6379
- `ray-client` pod: unchanged, still running

The validation command printed `ok` — the ray-client connected to the cluster successfully.

**What happened with the initial failure:** `ray.init(address='ray-head:6379')` in Ray 2.9.0 joins the cluster as a driver node, which requires spinning up a local raylet subprocess. On this kind cluster, that raylet takes longer than the 60-second wait window to register. The fix was to pre-start the ray worker on the client pod with `ray start --address=ray-head:6379` so the local raylet is already running when `ray.init` is called — it then connects instantly.
