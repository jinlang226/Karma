Done. Summary of what was changed and verified:

**Root cause:** The job script used `ray.init(address="ray://ray-head:10001")` — the Ray Client port, which the `ray-head` service does not expose (it only exposes `gcs:6379` and `dashboard:8265`), so the client connection timed out.

**Fix:**
- The script is mounted read-only from the `ray-job-script` ConfigMap (subPath `/opt/job.py`), so I patched the ConfigMap to use `ray.init(address="ray-head:6379")` and recreated the `ray-client` pod to pick up the new content.
- Connecting a driver via the GCS port (6379) requires the host to be a Ray node, so I joined the client pod to the cluster with `ray start --address=ray-head:6379` (the raylet persists for the pod's lifetime; `restartPolicy: Never`).

**Verification:**
```
kubectl -n ray exec ray-client -- python /opt/job.py
→ pong
EXIT: 0
```

The command connects to the head at `ray-head:6379`, prints `pong`, and exits 0.
