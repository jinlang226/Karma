## Summary

The 3-member MongoDB replica set (`rs0`) is now configured with split-horizon mappings, advertising external NodePort endpoints to external clients while keeping internal cluster DNS for in-cluster members.

**What I did:**
1. Identified the topology — mongod runs with `requireTLS`, primary was `mongo-rs-2`, and the NodePort service selectors map each pod to its external endpoint:
   - `mongo-rs-0` → `domain-rs-1:31181`
   - `mongo-rs-1` → `domain-rs-2:31182`

- `me = domain-rs-1:31181`
- `hosts = ["domain-rs-1:31181","domain-rs-2:31182","domain-rs-3:31183"]`
