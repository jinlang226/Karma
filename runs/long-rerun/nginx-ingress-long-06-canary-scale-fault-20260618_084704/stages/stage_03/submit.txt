Done. The canary routing now works correctly.

## Root cause
The `canary-canary` Ingress (host `canary.example.com`, backend `canary-echo`) had the canary trigger configured as:
```
nginx.ingress.kubernetes.io/canary-by-header-value: always
```
With an explicit `canary-by-header-value`, the `X-Canary` header must match that exact value to route to the canary backend. So `X-Canary: beta` did **not** match `always`, and all traffic fell through to the stable backend.

(The other `canary-alt` Ingress is a decoy — different host `canary-alt.example.com`, labeled `benchmark.decoy: "true"`, and points at `stable-echo`.)

## Fix
Changed the annotation on `canary-canary` to match the required header value:
```
nginx.ingress.kubernetes.io/canary-by-header-value: beta
```
No application Deployments or Services were touched — only the Ingress annotation.

## Verification
- No header → `stable` ✓
- `X-Canary: beta` → `canary` ✓ (stable across 3 repeated runs)
