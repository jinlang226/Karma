**What was wrong and what I fixed:**

1. **Missing backends**: The precondition probe passed early (curl-test was already present from a prior stage), so the `apply` steps never ran. Neither `stable-echo` nor `canary-echo` deployments/services existed. I applied them from the case resource files.

2. **Wrong ingress class**: The ingress-nginx controller in the cluster had been reconfigured (by a previous stage) to handle `ingress-1` class instead of `nginx`. The stable and canary ingresses were set to `ingressClassName: nginx`, which the running controller silently ignored. I patched both ingresses (`canary-stable` and `canary-canary`) to use `ingressClassName: ingress-1`, which the active controller handles.
