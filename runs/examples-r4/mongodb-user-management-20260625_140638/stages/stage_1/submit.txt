Everything is configured and verified. Here's a summary of what was done:

**Problem found:** MongoDB was running with `--auth --keyFile` but had no admin users bootstrapped, making it impossible to authenticate. The keyFile disables the localhost exception.

**Bootstrap approach:**
1. Scaled down the StatefulSet to 0
2. Temporarily patched the command to standalone mode (no `--auth`, no `--keyFile`, no `--replSet`) to access the data
3. Created and configured all users
4. Restored the original command and scaled back to 3 replicas

**Final configuration:**

| User | Auth DB | Roles |
|------|---------|-------|
| `admin-user` | admin | `userAdminAnyDatabase`, `readWriteAnyDatabase`, `clusterAdmin` (unchanged) |
| `app-user` | admin | `readWrite` on `appdb` ✓ |
| `readonly-user` | admin | `read` on `appdb` + `reportingRole` on `appdb` ✓ |

**`reportingRole`** (db: `appdb`):
- Privilege: `find` on `appdb.reports` (the `find` action enables `aggregate` in MongoDB)

**Verified:**
- `app-user` can insert, find, and delete on `appdb`
- `readonly-user` write is denied (Unauthorized)
- `readonly-user` `aggregate` on `appdb.reports` succeeds (returns 3 existing docs)
- All 3 replica set members healthy (PRIMARY + 2 SECONDARY), existing data intact
