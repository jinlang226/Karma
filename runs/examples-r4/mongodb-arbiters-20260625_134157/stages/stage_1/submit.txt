Done. Here's what was done and why it works:

**Problem:** The replica set had 2 voters (mongo-rs-0, mongo-rs-1). With only 2 votes, a majority requires both nodes — losing either one means no quorum, no election, no primary.

**Solution:** Added `mongo-arb-0` as an arbiter member:
- `arbiterOnly=true` — holds no data, no storage cost
- `votes=1` — provides the tiebreaking 3rd vote
- `priority=0` — can never become primary

**Result:** 3 votes total, majority = 2. Either data-bearing member can now fail and the surviving node can still win an election with the arbiter's vote.

**Prerequisite step:** MongoDB 5+ refuses to reconfig if it would silently change the implicit default write concern. Setting `defaultWriteConcern: {w: "majority"}` explicitly before `rs.addArb()` satisfies that guard.

**Data integrity:** `app.test` still has 3 documents — unchanged.
