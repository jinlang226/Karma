### How it was done (2-phase minimal-downtime rotation)

**Phase 1** — Added the new CA alongside the old CA in the trust bundle (`ca.crt` = old CA + new CA), kept the old leaf cert. Rolling restart → all nodes now trust both CAs. **Zero downtime** (old leaf signed by old CA, still trusted by everyone).

**Phase 2** — Replaced `ca.crt` with new CA only, and replaced `tls.crt`/`tls.key` with the new leaf cert signed by the new CA. Rolling restart. Brief ~30s disruption window during the middle pod's restart (expected: cluster briefly ran on 1 node while quorum reformed on the new-cert nodes).

**Client trust**: The `es-http-ca` configmap was updated with the new CA so clients can pick it up. HTTPS connectivity verified via `curl --cacert new-ca.crt` from the `curl-test` pod — cluster responds correctly.
