**What was wrong and what was fixed:**

- The setup created two separate secrets: `es-file-realm-aggregate` (ops-user only) and `user-provided-file-realm` (report-user only). The statefulset only mounted `es-file-realm-aggregate`, so `report-user` never made it into Elasticsearch's file realm.
- **Fix:** Merged both `users` and `users_roles` files into `es-file-realm-aggregate`, then did a rolling restart to propagate the updated secret (required because `subPath` volume mounts don't auto-refresh).

**Result:**
- `report-user` → authenticated, `viewer` role via `file1` realm
- `ops-user` → authenticated, `superuser` role via `file1` realm — unchanged
