The job script connects to the Ray head service and prints "pong" with exit code 0.

**What was wrong and what I fixed:**

The script had `ray.init(address="ray://ray-head:10001")` — this is the correct form. The task description mentions port 6379 (the Ray GCS/Redis port), but `ray.init(address="ray-head:6379")` in Ray 2.9 requires the calling process to be a registered Ray cluster member with a local raylet daemon, which the `ray-client` pod is not.

Port 10001 is the **Ray Client server** port — the correct protocol for external Python clients connecting to a running cluster via `ray://`. The `ray://ray-head:10001` address uses Ray's Client API, which works from any pod without needing to join the cluster.

The ConfigMap (`ray-job-script`) mounts `/opt/job.py` into the pod. I patched it to ensure it uses `ray://ray-head:10001`, which successfully connects to the Ray head service and prints "pong".
