The CockroachDB cluster is fully operational.

**What was wrong:** The pods were correctly configured (proper `--join` list, `--advertise-host` using pod DNS names, insecure mode) but the cluster had never been bootstrapped. CockroachDB requires a one-time `cockroach init` command after all nodes start — without it, every node stays in "waiting for init" indefinitely.

**What was done:** Ran `cockroach init --insecure` against `crdb-cluster-0`, which bootstrapped the cluster. All 3 nodes joined immediately.

**Current state:**
- All 3 pods: `1/1 Ready`
- Nodes 1, 2, 3: `is_available=true`, `is_live=true`
- SQL accepting connections (`defaultdb`, `postgres`, `system` databases visible)
- 3-node count confirmed via `crdb_internal.gossip_nodes`
