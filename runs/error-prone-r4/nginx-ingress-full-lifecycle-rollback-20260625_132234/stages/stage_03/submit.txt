Both routing rules work correctly.

**Root cause:** The controller in the `ingress-nginx` namespace had been reconfigured (via duplicate `--controller-class`/`--ingress-class` args) to identify as `k8s.io/ingress-nginx-1`, handling the `ingress-1` IngressClass. But `canary-stable` and `canary-canary` were pointing at `ingressClassName: nginx`, whose controller (`k8s.io/ingress-nginx`) had no running instance — so the controller silently ignored both Ingresses, producing 404s.

**Fix:** Patched both `canary-stable` and `canary-canary` to use `ingressClassName: ingress-1`, which the active controller in `ingress-nginx` namespace actually handles. The canary annotations (`nginx.ingress.kubernetes.io/canary: "true"`, `canary-by-header: X-Canary`, `canary-by-header-value: always`) were already correct on `canary-canary` — they just needed to reach an active controller.
