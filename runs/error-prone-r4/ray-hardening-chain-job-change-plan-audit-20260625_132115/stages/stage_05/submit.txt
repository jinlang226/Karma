2. **Fixed the script**: The service only exposes port 6379 (GCS). Connecting via `ray.init(address="ray-head:6379")` requires a local Ray daemon to be running (with session files). The updated script mirrors what worker pods do — it runs `ray start --address=ray-head:6379` via subprocess to join the cluster, then calls `ray.init(address="auto")` which finds the local session.

3. **Updated the ConfigMap** and recreated the pod (required because `subPath` mounts don't auto-refresh on ConfigMap changes).
