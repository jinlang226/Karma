The 3-node Elasticsearch cluster is fully deployed and operational:

- **All 3 nodes Ready** (`es-0`, `es-1`, `es-2` — 1/1 each)
- **Cluster status: green** — 3 nodes, 3 data nodes, no unassigned shards
- **es-http Service** reachable at port 9200 over HTTPS with TLS from the existing `es-http-tls` secret
- **Reads and writes verified** — wrote and retrieved a document via the `es-http` service

Two non-obvious issues resolved during deployment:
1. `podManagementPolicy: Parallel` — needed so all 3 pods launch simultaneously (otherwise the StatefulSet waits for es-0 to be Ready before creating es-1/es-2, creating a deadlock)
2. `publishNotReadyAddresses: true` on the headless `es-nodes` service — Kubernetes only adds not-yet-Ready pods to headless service DNS when this is set; without it, pods couldn't resolve each other's addresses and the quorum was never formed
