Everything is working correctly:

- **HTTPS at `es.example.com`**: Returns Elasticsearch API responses (cluster status green, 3 nodes).
- **Plain HTTP**: Returns `308 Permanent Redirect` to `https://es.example.com` — the request does not succeed with data.
- **Cluster health**: Green, all 3 nodes and pods running.

**What was done:**
1. Generated a self-signed TLS certificate for `es.example.com` using the existing `openssl-toolbox` pod.
2. Created a `es-tls` Secret in the `elasticsearch` namespace with that cert/key.
3. Created an `Ingress` (`es-ingress`) using `ingressClassName: nginx` with:
   - TLS termination via the `es-tls` secret
   - `ssl-redirect` and `force-ssl-redirect: "true"` annotations so HTTP requests are permanently redirected (308) to HTTPS and never served content directly
   - Routes `es.example.com /` → `es-http:9200` (plain HTTP backend)
