All three pods are `1/1 Ready` and the replica set shows:
- `mongodb-replica-0`: **PRIMARY** (health: 1)
- `mongodb-replica-1`: **SECONDARY** (health: 1)
- `mongodb-replica-2`: **SECONDARY** (health: 1)

**Root cause:** The `health-overrides` ConfigMap had a stale entry for `mongodb-replica-1` with the password `legacyHealthPass123`, but the actual MongoDB `health-user` password is `healthpass123`. The health-check script on that pod was picking up the wrong override password, causing auth failures on every probe.

**Fix:** Removed the `mongodb-replica-1: legacyHealthPass123` entry from the `health-overrides` ConfigMap, so the script now falls back to the correct password from the `health-user-password` secret.
