All three members are healthy:

- **mongodb-replica-0**: PRIMARY (health=1), Ready 1/1
- **mongodb-replica-1**: SECONDARY (health=1), Ready 1/1  
- **mongodb-replica-2**: SECONDARY (health=1), Ready 1/1

**Root cause:** The `health-overrides` ConfigMap had an incorrect/stale password entry (`legacyHealthPass123`) for `mongodb-replica-1`. The health check script reads pod-name-keyed overrides and used this wrong password instead of the correct `healthpass123` from the `health-user-password` secret, causing authentication failures on every readiness probe.

**Fix applied:** Removed the incorrect `mongodb-replica-1` key from the `health-overrides` ConfigMap. Once the ConfigMap propagated to the pod (~1 minute), the next liveness probe failure triggered a container restart. After restart, the health check used the correct password from the secret and both probes passed, restoring the pod to Ready state.
