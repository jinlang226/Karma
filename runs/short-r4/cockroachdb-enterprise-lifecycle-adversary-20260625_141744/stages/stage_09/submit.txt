Both are working correctly:

- **HTTPS DB Console**: returns HTTP 200 with a valid TLS cert (CN=crdb-ui.example.com, SAN=DNS:crdb-ui.example.com)
- **TCP port 26257**: the connection goes through and reaches CockroachDB (responding with its node TLS cert CN=node, O=CockroachDB)

Here's a summary of what was configured:

| What | Resource | Detail |
|------|----------|--------|
| TLS cert | `Secret/crdb-ui-tls` in `cockroachdb` | Self-signed for `crdb-ui.example.com` |
| DB Console HTTPS | `Ingress/crdb-ui` in `cockroachdb` | `ingressClassName: nginx`, TLS termination, `backend-protocol: HTTPS` → `crdb-cluster-public:8080` |
| TCP SQL routing | `ConfigMap/tcp-services` in `ingress-nginx` | `"26257": cockroachdb/crdb-cluster-public:26257` |
| Controller flag | `Deployment/ingress-nginx-controller` | `--tcp-services-configmap=ingress-nginx/tcp-services` added |
| Service port | `Service/ingress-nginx-controller` | Port 26257 (TCP) added, named `sql-cockroachdb` |

The DB Console is reachable at `https://crdb-ui.example.com` (TLS terminated at nginx, proxied over HTTPS to CRDB's port 8080), and CockroachDB SQL is accessible via `<ingress-ip>:26257` (nginx TCP proxy to `crdb-cluster-public:26257`).
