The command exits 0 and prints "pong". Here's what was done:

**Root cause:** The original script used `ray://ray-head:10001` (Ray Client protocol), but port 10001 is not exposed by the `ray-head` Service — only port 6379 (GCS) and 8265 (dashboard) are.

**Fix:** The updated script now:
1. Runs `ray start --address=ray-head:6379 --num-cpus=0` to join the Ray cluster as a zero-CPU worker node at the pod's own IP
2. Calls `ray.init(address="ray-head:6379")` — which works once the local raylet is registered with GCS
3. Prints "pong" and cleans up with `ray stop`

The ConfigMap `ray-job-script` was patched with the new script, and the pod was recreated so the `subPath`-mounted file at `/opt/job.py` picked up the change.
