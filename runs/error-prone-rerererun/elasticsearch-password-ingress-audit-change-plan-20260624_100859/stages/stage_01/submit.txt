Everything is working. Here's a summary of what was deployed:

**Resources created:**
- `ConfigMap/elasticsearch-config` — ES config with security + HTTP/transport TLS using the existing `es-http-tls` secret
- `Service/elasticsearch-headless` — headless (ClusterIP: None) with `publishNotReadyAddresses: true` so nodes can discover each other via DNS before readiness passes
- `Service/es-http` — ClusterIP service on port 9200 named as required
- `StatefulSet/elasticsearch` — 3 replicas, `podManagementPolicy: Parallel` so all nodes start simultaneously, using `ELASTIC_PASSWORD` from the `elastic-password` secret
- `PodDisruptionBudget/elasticsearch-pdb` — minAvailable: 2

**Key issues resolved during deployment:**
1. `ELASTIC_PASSWORD` is an env var (not a `elasticsearch.yml` key)
2. ES 8.x requires transport SSL enabled when security is on — used the same `es-http-tls` certs for transport
3. `podManagementPolicy: Parallel` prevents the deadlock where pod-0 can't be Ready without pod-1/2 and vice versa
4. `publishNotReadyAddresses: true` on the headless service allows inter-node DNS resolution before pods pass readiness

**Result:** 3-node cluster, status green, reads and writes confirmed through `es-http:9200` over HTTPS.
