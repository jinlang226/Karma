The command exits 0 and prints `pong`. Here's what was done:

**Root cause:** `ray-head:6379` is the GCS (cluster metadata) port. `ray.init(address="ray-head:6379")` tries to join the cluster as a worker node, which requires a local Ray process to already be running on the client pod. The original script was using port 10001 (Ray Client Server protocol), which is the correct remote-client approach — but the task required using port 6379.

**Fix:** Updated `/opt/job.py` (via the `ray-job-script` ConfigMap) to:
1. `ray start --address=ray-head:6379` — starts a local Ray worker on the client pod and joins it to the cluster
2. `ray.init()` — connects the Python driver to that local worker
3. Prints `pong`, then cleanly shuts down and stops the local worker
