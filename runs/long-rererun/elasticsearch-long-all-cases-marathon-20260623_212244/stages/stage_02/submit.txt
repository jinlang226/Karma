The deployment is complete and fully functional. Here's a summary of what was done:

**Problem with previous state:** An ES 7.17.9 cluster was already deployed using the wrong image version, with no TLS or authentication, and the required secrets didn't exist.

**What was deployed:**

1. **TLS certificates** — Generated a CA and server cert with SANs covering all service/pod DNS names, stored in secret `es-http-tls`.

2. **Password secret** — Created `elastic-password` with the bootstrap password for the `elastic` user.

3. **ConfigMap `es-config`** — ES 8.11.1 config with xpack security + TLS enabled for both HTTP and transport layers.

4. **Services** — Headless `es-cluster` (port 9300, `publishNotReadyAddresses: true` to break the DNS bootstrap deadlock) and ClusterIP `es-http` (port 9200).

5. **StatefulSet** — 3 replicas with `podManagementPolicy: Parallel` (so all 3 pods start simultaneously and can form the cluster), image `docker.elastic.co/elasticsearch/elasticsearch:8.11.1`.

**Result:**
- All 3 nodes: `1/1 Ready`
- Cluster health: **green**, 3/3 nodes
- HTTPS reads and writes through `es-http:9200`: **working**
