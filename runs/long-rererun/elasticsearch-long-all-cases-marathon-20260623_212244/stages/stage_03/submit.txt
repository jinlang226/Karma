All users authenticate successfully and the cluster remains healthy. Here's what was done:

**Root cause:** `report-user` was added to a separate secret (`user-provided-file-realm`) instead of the `es-file-realm-aggregate` secret that the cluster reads. Additionally, the statefulset wasn't mounting any file-realm secret at all, so no file-realm users could authenticate.

**Fixes applied:**

1. **Merged users** — combined `report-user` credentials from `user-provided-file-realm` into `es-file-realm-aggregate` (which already held `ops-user`)

2. **Patched the statefulset** — added a `file-realm` volume backed by `es-file-realm-aggregate` and mounted `users` and `users_roles` at `/usr/share/elasticsearch/config/users` and `/usr/share/elasticsearch/config/users_roles` — the paths ES reads by default for the file realm

3. **Created the `read` role** — `read` is not a built-in role in ES 7.17.9 (unlike other versions), so `report-user`'s role assignment resolved to nothing; created it via the Security API with index-level read privileges

4. **Seeded `app-data` index** — the precondition couldn't seed it earlier (ops-user wasn't auth'd yet); created it now so report-user has data to read
