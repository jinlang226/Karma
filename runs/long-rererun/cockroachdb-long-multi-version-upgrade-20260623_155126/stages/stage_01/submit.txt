The 3-node CockroachDB cluster is fully deployed and healthy. Here's what was created:

**Resources deployed:**
- `Service/crdb-cluster` — headless service (ClusterIP: None) for pod DNS, exposing grpc/26257 and http/8080
- `Service/crdb-cluster-public` — standard ClusterIP service for SQL/admin access
- `StatefulSet/crdb-cluster` — 3 replicas, `podManagementPolicy: Parallel`, image `cockroachdb/cockroach:v23.2.0`, each advertising its pod DNS (e.g. `crdb-cluster-0.crdb-cluster.cockroachdb.svc.cluster.local`)
- `PodDisruptionBudget/crdb-cluster-pdb` — `minAvailable: 2`, ensuring at most 1 voluntary disruption at a time (quorum preserved)
- 3 PVCs bound at 10Gi each

**Key detail:** The StatefulSet uses `podManagementPolicy: Parallel` so all 3 pods start simultaneously (required since each node waits for peers before accepting `cockroach init`). After all pods reached Running state, `cockroach init` was run once to bootstrap the cluster. All 3 nodes report `is_available=true, is_live=true`.
