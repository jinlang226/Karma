All done. Here's what was wrong and what was fixed:

**Root cause:** The `health-overrides` ConfigMap contained a stale entry `mongodb-replica-1: legacyHealthPass123`. The health check script mounts this ConfigMap and, for pod `mongodb-replica-1`, reads from the override file instead of the main secret. Since the actual `health-user` password is `healthpass123` (from the `health-user-password` Secret), pod 1's probes were sending the wrong credentials — causing repeated `MongoServerError: Authentication failed` for both readiness and liveness probes.

**Fix:** Patched `health-overrides` to remove the stale `mongodb-replica-1` key. The pod now falls back to reading the password from the Secret mount, authentication succeeds, and all probes pass.
