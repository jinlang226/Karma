HTTPS is restored. The curl returns exit code 0 and prints "hello".

**Root cause:** The Ingress secret `expired-tls-secret` contained a TLS certificate valid only Jan 1–2, 2020. The certificate was signed by the test CA (`/tmp/test-ca.key`).

**Fix:** Generated a new RSA-2048 certificate for `demo.example.com` with a 365-day validity (Jun 2026–Jun 2027), signed by the same test CA, and replaced the secret contents via `kubectl apply`. The nginx ingress controller picked up the new secret within seconds, and TLS handshakes now succeed with `SSL certificate verify ok`.
